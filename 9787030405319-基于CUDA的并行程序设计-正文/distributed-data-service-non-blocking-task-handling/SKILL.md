# 分布式数据服务非阻塞任务处理

## 何时使用
- 节点收到数据请求但目标数据不在本地内存中
- 系统要求高吞吐、低延迟且避免死锁
- 多节点协同环境中需防止请求堆积和重复网络请求
- 内存资源受限，需动态管理数据缓存

## 执行步骤

### 1. 统一监听入口
- 所有功能（地址请求、数据请求、外存读写等）完成后立即返回同一监听端口
- 持续监听新请求，确保服务不间断

### 2. 非阻塞处理逻辑
- 当所需数据不在内存中时，立即触发“外存查询”或“向外请求”，然后返回监听端口
- 不等待响应结果，避免阻塞当前进程
- 防止节点间互相等待导致死锁

### 3. 关键功能流程
- **地址请求处理**：
  - 检查全局数组表
  - 若无记录 → 触发“向外请求”
  - 若有记录但数据不在内存 → 触发“外存查询”
  - 若数据已在内存 → 直接返回地址
- **外存查询**：
  - 检查外存统计文件
  - 若存在 → 触发“外存读写”
  - 否则 → 触发“向外请求”
- **防重复请求**：
  - 向外请求前检查“向外请求记录文件”
  - 若已存在相同请求，跳过发送
- **数据接收后处理**：
  - 触发“检查工作”流程
  - 主动满足此前等待该数据的请求

### 4. 内存管理
- 实时监控内存与GPU显存使用状态
- 接近临界点时，按LRU（最近最久未使用）策略销毁数据块
- 保留全局/块数组的元信息，确保可重新加载

### 5. 系统关闭协调
- 进入“预备注销”状态
- 确保所有节点完成当前任务并协调一致关闭

> 注意：本机制依赖“向外请求记录文件”避免带宽浪费，且不适用于需要强一致性的同步阻塞场景。
