# 4.3 共享存储器的使用技巧

## 一、本节作用

本节用于整理 CUDA 共享存储器 shared memory 的使用方法和优化技巧。

在 CUDA 存储器层次中，shared memory 位于 GPU 片上，访问速度通常明显快于 global memory。它由同一个 block 内的所有线程共享，常用于缓存 global memory 中会被重复访问的数据，或者用于 block 内线程之间的协作。

shared memory 是 CUDA 性能优化中最常用、最重要的存储器之一。它经常出现在以下场景中：

1. 矩阵乘法分块。
2. 并行归约。
3. 图像滤波。
4. stencil 计算。
5. 矩阵转置。
6. block 内线程协作。
7. global memory 访问重排。
8. 减少重复 global memory 读取。

掌握本节后，应能够完成以下任务：

1. 正确声明和使用 shared memory。
2. 理解 shared memory 的 block 内共享机制。
3. 正确使用 `__syncthreads()`。
4. 判断什么时候应该使用 shared memory。
5. 分析 shared memory bank conflict。
6. 使用 padding 避免二维数组列访问冲突。
7. 使用 shared memory 优化矩阵乘法。
8. 使用 shared memory 实现并行归约。
9. 使用 shared memory 优化图像滤波。
10. 避免 shared memory 使用中的常见错误。

## 二、shared memory 的基本特点

shared memory，即共享存储器，是 GPU 片上的高速存储器。

它具有以下特点：

1. 位于 GPU 片内。
2. 访问速度快。
3. 容量较小。
4. 同一个 block 内线程共享。
5. 不同 block 之间不能共享同一块 shared memory。
6. 生命周期与 block 相同。
7. 适合缓存可复用数据。
8. 适合实现 block 内线程协作。
9. 使用时通常需要配合 `__syncthreads()`。
10. 访问模式不合理时可能产生 bank conflict。

shared memory 的核心价值是：

```text
用较快的片上存储器缓存 global memory 中会被重复使用的数据，从而减少慢速 global memory 访问。
```

---

## 三、shared memory 与 global memory 的关系

global memory 容量大，但访问延迟高。shared memory 容量小，但访问速度快。

二者常见关系如下：

```text
global memory 保存完整输入输出数据
shared memory 缓存当前 block 需要重复使用的一小块数据
```

典型流程：

```text
1. 每个 block 从 global memory 读取一块数据
2. 将数据写入 shared memory
3. 使用 __syncthreads() 等待所有线程完成加载
4. block 内线程从 shared memory 中反复读取数据进行计算
5. 将结果写回 global memory
```

示意代码：

```cpp
__global__ void kernel(const float* input, float* output, int n) {
    __shared__ float tile[256];

    int tid = threadIdx.x;
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        tile[tid] = input[idx];
    }

    __syncthreads();

    if (idx < n) {
        output[idx] = tile[tid] * 2.0f;
    }
}
```

在这个示例中，`tile` 是 shared memory。它先缓存从 global memory 读取的数据，然后由同一个 block 内的线程使用。

## 四、shared memory 的声明方式

CUDA 中 shared memory 主要有两种声明方式：

1. 静态 shared memory。
2. 动态 shared memory。

---

## 五、静态 shared memory

静态 shared memory 的大小在编译期确定。

声明方式：

```cpp
__shared__ float tile[256];
```

示例：

```cpp
__global__ void staticSharedKernel(const float* input, float* output, int n) {
    __shared__ float tile[256];

    int tid = threadIdx.x;
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        tile[tid] = input[idx];
    } else {
        tile[tid] = 0.0f;
    }

    __syncthreads();

    if (idx < n) {
        output[idx] = tile[tid];
    }
}
```

静态 shared memory 的特点：

1. 写法简单。
2. 数组大小固定。
3. 适合 block 大小固定的场景。
4. 编译器可以更容易分析内存使用情况。

---

## 六、动态 shared memory

动态 shared memory 的大小在 kernel 启动时指定。

声明方式：

```cpp
extern __shared__ float tile[];
```

kernel 启动时通过第三个参数指定 shared memory 大小：

```cpp
kernel<<<grid, block, sharedMemSize>>>(...);
```

示例：

```cpp
__global__ void dynamicSharedKernel(const float* input, float* output, int n) {
    extern __shared__ float tile[];

    int tid = threadIdx.x;
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        tile[tid] = input[idx];
    } else {
        tile[tid] = 0.0f;
    }

    __syncthreads();

    if (idx < n) {
        output[idx] = tile[tid];
    }
}
```

启动：

```cpp
dim3 block(256);
dim3 grid((n + block.x - 1) / block.x);

size_t sharedMemSize = block.x * sizeof(float);

dynamicSharedKernel<<<grid, block, sharedMemSize>>>(d_input, d_output, n);
```

动态 shared memory 的特点：

1. 大小可以在运行时决定。
2. 适合 block 大小或数据块大小可变的场景。
3. 需要在 kernel 启动时显式传入字节数。
4. 多个动态 shared memory 数组需要手动划分同一片内存。

---

## 七、多个动态 shared memory 数组的划分

如果需要在一个 kernel 中使用多个动态 shared memory 数组，可以在同一个 `extern __shared__` 缓冲区中手动划分。

示例：

```cpp
__global__ void multiSharedKernel(float* output, int n) {
    extern __shared__ float shared[];

    float* s_A = shared;
    float* s_B = shared + blockDim.x;

    int tid = threadIdx.x;

    s_A[tid] = 1.0f;
    s_B[tid] = 2.0f;

    __syncthreads();

    int idx = blockIdx.x * blockDim.x + tid;

    if (idx < n) {
        output[idx] = s_A[tid] + s_B[tid];
    }
}
```

启动时需要分配两个数组的空间：

```cpp
size_t sharedMemSize = 2 * block.x * sizeof(float);

multiSharedKernel<<<grid, block, sharedMemSize>>>(d_output, n);
```

如果多个数组类型不同，需要注意对齐问题。例如先放 `float`，再放 `int` 时，应确保地址满足类型对齐要求。

## 八、block 内线程共享机制

shared memory 的作用范围是 block。

这意味着：

```text
同一个 block 内的所有线程可以访问同一块 shared memory
不同 block 有各自独立的 shared memory
```

例如：

```cpp
__global__ void kernel() {
    __shared__ float s_data[256];

    s_data[threadIdx.x] = threadIdx.x;
}
```

如果 kernel 启动了 100 个 block，那么每个 block 都会有自己独立的 `s_data[256]`。不同 block 的 `s_data` 不是同一块存储器。

因此：

1. shared memory 可以用于同一 block 内线程通信。
2. shared memory 不能用于不同 block 之间通信。
3. 不同 block 之间如果需要共享数据，通常应使用 global memory。
4. 不同 block 之间如果需要全局同步，通常需要拆成多个 kernel。

## 九、`__syncthreads()` 同步

## 9.1 `__syncthreads()` 的作用

`__syncthreads()` 用于同步同一个 block 内的所有线程。

它的作用是：

```text
等待同一个 block 内所有线程都执行到该语句后，再继续向下执行。
```

在 shared memory 编程中，`__syncthreads()` 非常重要。

典型使用场景：

```cpp
__shared__ float tile[256];

int tid = threadIdx.x;
tile[tid] = input[tid];

__syncthreads();

float value = tile[(tid + 1) % 256];
```

如果没有 `__syncthreads()`，某些线程可能在其他线程尚未写入 shared memory 时就开始读取，导致结果不确定。

---

## 9.2 什么时候需要 `__syncthreads()`

需要同步的典型情况：

1. 多个线程先写 shared memory，之后多个线程再读 shared memory。
2. block 内归约每一轮计算之后。
3. 矩阵乘法中每一轮 tile 加载完成之后。
4. 图像滤波中局部 tile 和 halo 区域加载完成之后。
5. shared memory 中的数据会被其他线程读取。

不需要同步的情况：

1. 每个线程只读写自己独立的 shared memory 元素，不被其他线程读取。
2. shared memory 只是临时缓存，线程之间没有依赖。
3. warp 内使用更底层的 warp-level 同步机制时，但这属于更高级主题。

---

## 9.3 `__syncthreads()` 的错误用法

错误示例：

```cpp
if (threadIdx.x == 0) {
    __syncthreads();
}
```

这是错误的，因为只有部分线程执行到 `__syncthreads()`，其他线程没有执行到，可能导致 block 内线程永久等待。

另一个错误示例：

```cpp
if (threadIdx.x < 128) {
    __syncthreads();
}
```

同样错误，因为不是所有线程都会进入这个分支。

正确原则：

```text
同一个 block 内的所有线程必须以一致的控制流执行到同一个 __syncthreads()。
```

可以这样写：

```cpp
if (threadIdx.x < 128) {
    // do something
}

__syncthreads();

if (threadIdx.x < 128) {
    // do something after synchronization
}
```

## 十、bank conflict

## 10.1 bank 的基本概念

shared memory 被划分为多个 bank。多个线程同时访问 shared memory 时，如果访问分布在不同 bank 中，就可以并行完成；如果多个线程访问同一个 bank 中的不同地址，就可能产生 bank conflict。

bank conflict 会导致访问被串行化，从而降低 shared memory 性能。

简单理解：

```text
同一 warp 内多个线程访问不同 bank：效率高
同一 warp 内多个线程访问同一 bank 的不同地址：发生 bank conflict
```

---

## 10.2 bank conflict 的典型例子

假设声明二维 shared memory：

```cpp
__shared__ float tile[32][32];
```

如果按行访问：

```cpp
float value = tile[row][threadIdx.x];
```

同一个 warp 中相邻线程访问的是同一行中的连续元素，通常分布在不同 bank 中，访问效率较高。

但如果按列访问：

```cpp
float value = tile[threadIdx.x][col];
```

同一个 warp 中相邻线程访问的是不同行的同一列。由于二维数组按行连续存储，每一行长度正好是 32 个 `float`，因此同一列元素可能映射到同一个 bank。

示意：

```text
tile[0][0] -> bank 0
tile[1][0] -> bank 0
tile[2][0] -> bank 0
...
tile[31][0] -> bank 0
```

此时同一个 warp 中多个线程访问同一个 bank 的不同地址，会发生严重 bank conflict。

---

## 10.3 padding 避免 bank conflict

解决二维 shared memory 列访问冲突的常用方法是在每行末尾增加一个额外元素。

原始写法：

```cpp
__shared__ float tile[32][32];
```

优化写法：

```cpp
__shared__ float tile[32][33];
```

也就是：

```cpp
__shared__ float tile[32][32 + 1];
```

这样每一行的长度从 32 变成 33，下一行起始位置会相对 bank 偏移一个位置。

此时同一列元素的 bank 分布变为：

```text
tile[0][0] -> bank 0
tile[1][0] -> bank 1
tile[2][0] -> bank 2
...
tile[31][0] -> bank 31
```

这样同一 warp 访问同一列时就可以避免多个线程集中访问同一个 bank。

---

## 10.4 padding 的代价

padding 会增加 shared memory 使用量。

例如：

```cpp
__shared__ float tile[32][32];
```

大小为：

```text
32 × 32 × 4B = 4096B
```

加入 padding 后：

```cpp
__shared__ float tile[32][33];
```

大小为：

```text
32 × 33 × 4B = 4224B
```

增加了：

```text
128B
```

通常这个代价很小，但对于大型 tile 或多个 shared memory 数组，仍需要注意 shared memory 总占用量。

---

## 10.5 broadcast 情况

如果同一个 warp 中多个线程访问 shared memory 的同一个地址，现代 GPU 通常可以通过广播机制处理，不一定产生普通意义上的 bank conflict。

例如：

```cpp
float value = tile[0];
```

如果一个 warp 中所有线程都读取 `tile[0]`，这通常可以被高效广播。

需要区分：

```text
多个线程访问同一 bank 的不同地址：可能 bank conflict
多个线程访问完全相同地址：可能广播，不一定冲突
```

## 十一、shared memory 在矩阵乘法中的应用

## 11.1 基础矩阵乘法的问题

矩阵乘法：

```text
C = A × B
```

其中：

```cpp
C[row][col] = sum(A[row][k] * B[k][col])
```

基础 CUDA 实现中，每个线程计算 `C` 的一个元素：

```cpp
__global__ void matMulBasic(const float* A,
                            const float* B,
                            float* C,
                            int N) {
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    int row = blockIdx.y * blockDim.y + threadIdx.y;

    if (row < N && col < N) {
        float sum = 0.0f;

        for (int k = 0; k < N; ++k) {
            sum += A[row * N + k] * B[k * N + col];
        }

        C[row * N + col] = sum;
    }
}
```

这个版本的问题是：

1. 每个线程都从 global memory 重复读取 A 和 B。
2. 相邻线程会重复访问相同的 A 行数据。
3. 相邻线程会重复访问相同的 B 列数据。
4. global memory 访问次数多，性能受限。

---

## 11.2 shared memory 分块思想

shared memory 优化矩阵乘法的核心思想是：

```text
把 A 和 B 的小块 tile 从 global memory 加载到 shared memory 中；
同一个 block 内多个线程重复使用这些 tile；
每轮计算完成后再加载下一块 tile。
```

这样可以显著减少 global memory 重复访问。

---

## 11.3 tiled matrix multiplication 示例

```cpp
#define TILE_SIZE 16

__global__ void matMulTiled(const float* A,
                            const float* B,
                            float* C,
                            int N) {
    __shared__ float s_A[TILE_SIZE][TILE_SIZE];
    __shared__ float s_B[TILE_SIZE][TILE_SIZE];

    int tx = threadIdx.x;
    int ty = threadIdx.y;

    int row = blockIdx.y * TILE_SIZE + ty;
    int col = blockIdx.x * TILE_SIZE + tx;

    float sum = 0.0f;

    int numTiles = (N + TILE_SIZE - 1) / TILE_SIZE;

    for (int t = 0; t < numTiles; ++t) {
        int tiledCol = t * TILE_SIZE + tx;
        int tiledRow = t * TILE_SIZE + ty;

        if (row < N && tiledCol < N) {
            s_A[ty][tx] = A[row * N + tiledCol];
        } else {
            s_A[ty][tx] = 0.0f;
        }

        if (tiledRow < N && col < N) {
            s_B[ty][tx] = B[tiledRow * N + col];
        } else {
            s_B[ty][tx] = 0.0f;
        }

        __syncthreads();

        for (int k = 0; k < TILE_SIZE; ++k) {
            sum += s_A[ty][k] * s_B[k][tx];
        }

        __syncthreads();
    }

    if (row < N && col < N) {
        C[row * N + col] = sum;
    }
}
```

---

## 11.4 矩阵乘法中两次同步的意义

在 tiled matrix multiplication 中，每轮循环通常需要两次同步。

第一次同步：

```cpp
__syncthreads();
```

位于 tile 加载之后，确保 `s_A` 和 `s_B` 中的数据都已经加载完成，所有线程才能开始计算。

第二次同步：

```cpp
__syncthreads();
```

位于 tile 计算之后，确保所有线程已经使用完当前 tile，然后才能进入下一轮加载，避免某些线程提前覆盖 shared memory。

## 十二、shared memory 在并行归约中的应用

## 12.1 并行归约的基本思想

并行归约 reduction 常用于求和、求最大值、求最小值等操作。

例如，对数组求和：

```text
sum = a[0] + a[1] + ... + a[n-1]
```

在 CUDA 中，常用做法是：

1. 每个 block 处理一段输入数据。
2. 每个线程从 global memory 读取一个或多个元素。
3. 将数据放入 shared memory。
4. block 内进行树形归约。
5. 每个 block 输出一个部分和到 global memory。
6. 再对部分和继续归约。

---

## 12.2 基础 shared memory 归约示例

```cpp
__global__ void reduceSumKernel(const float* input, float* blockSums, int n) {
    extern __shared__ float s_data[];

    unsigned int tid = threadIdx.x;
    unsigned int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        s_data[tid] = input[idx];
    } else {
        s_data[tid] = 0.0f;
    }

    __syncthreads();

    for (unsigned int stride = blockDim.x / 2; stride > 0; stride >>= 1) {
        if (tid < stride) {
            s_data[tid] += s_data[tid + stride];
        }

        __syncthreads();
    }

    if (tid == 0) {
        blockSums[blockIdx.x] = s_data[0];
    }
}
```

启动方式：

```cpp
dim3 block(256);
dim3 grid((n + block.x - 1) / block.x);

size_t sharedMemSize = block.x * sizeof(float);

reduceSumKernel<<<grid, block, sharedMemSize>>>(d_input, d_blockSums, n);
```

---

## 12.3 归约中的同步

归约中每一轮计算之后都需要同步：

```cpp
__syncthreads();
```

原因是下一轮归约依赖上一轮所有线程写入 shared memory 的结果。

如果省略同步，某些线程可能读取到尚未更新完成的数据。

---

## 12.4 归约中的优化方向

并行归约可以继续优化：

1. 每个线程读取两个元素，减少 block 数量。
2. 避免低效求模运算。
3. 减少分支发散。
4. 最后一个 warp 使用 warp-level 优化。
5. 使用 sequential addressing 减少 bank conflict。
6. 使用模板参数固定 block size。
7. 对多个 block 的输出再次归约。

本文件主要强调 shared memory 的基本作用；更深入的归约优化可放在 `5.6_optimization_examples.md` 中整理。

## 十三、shared memory 在图像滤波中的应用

## 13.1 图像滤波中的重复访问问题

图像滤波中，每个输出像素通常需要读取输入图像邻域。

例如 3×3 均值滤波中，每个像素需要读取周围 9 个像素：

```text
左上  上  右上
左    中  右
左下  下  右下
```

如果每个线程都直接从 global memory 读取 3×3 邻域，那么相邻线程之间会重复读取大量相同像素。

---

## 13.2 使用 shared memory 缓存图像 tile

优化思想：

```text
每个 block 先把对应图像区域加载到 shared memory；
再从 shared memory 中读取邻域进行滤波。
```

对于 3×3 滤波，如果 block 负责 16×16 个输出像素，则 shared memory 通常需要加载包含 halo 的 18×18 区域。

示意：

```text
输出区域：16 × 16
需要输入区域：(16 + 2) × (16 + 2)
```

---

## 13.3 简化版 3×3 滤波 shared memory 示例

下面示例展示 shared memory 图像滤波的基本结构。为简化代码，只处理非边界像素。

```cpp
#define BLOCK_X 16
#define BLOCK_Y 16
#define RADIUS 1

__global__ void meanFilter3x3Shared(const float* input,
                                    float* output,
                                    int width,
                                    int height) {
    __shared__ float tile[BLOCK_Y + 2 * RADIUS][BLOCK_X + 2 * RADIUS];

    int tx = threadIdx.x;
    int ty = threadIdx.y;

    int x = blockIdx.x * BLOCK_X + tx;
    int y = blockIdx.y * BLOCK_Y + ty;

    int sharedX = tx + RADIUS;
    int sharedY = ty + RADIUS;

    if (x < width && y < height) {
        tile[sharedY][sharedX] = input[y * width + x];
    } else {
        tile[sharedY][sharedX] = 0.0f;
    }

    if (tx < RADIUS && x > 0 && y < height) {
        tile[sharedY][sharedX - RADIUS] = input[y * width + (x - RADIUS)];
    }

    if (tx >= BLOCK_X - RADIUS && x + RADIUS < width && y < height) {
        tile[sharedY][sharedX + RADIUS] = input[y * width + (x + RADIUS)];
    }

    if (ty < RADIUS && y > 0 && x < width) {
        tile[sharedY - RADIUS][sharedX] = input[(y - RADIUS) * width + x];
    }

    if (ty >= BLOCK_Y - RADIUS && y + RADIUS < height && x < width) {
        tile[sharedY + RADIUS][sharedX] = input[(y + RADIUS) * width + x];
    }

    __syncthreads();

    if (x > 0 && x < width - 1 && y > 0 && y < height - 1) {
        float sum = 0.0f;

        for (int dy = -1; dy <= 1; ++dy) {
            for (int dx = -1; dx <= 1; ++dx) {
                sum += tile[sharedY + dy][sharedX + dx];
            }
        }

        output[y * width + x] = sum / 9.0f;
    }
}
```

这个示例展示了 shared memory 图像滤波的基本思想，但实际工程中还需要更严谨地处理四个角的 halo 区域和图像边界。

## 十四、shared memory 在矩阵转置中的应用

矩阵转置是 shared memory 经典应用之一。

基础转置中，如果直接从 global memory 按行读取、按列写回，写操作可能不连续，导致 global memory 访问效率低。

优化思想：

```text
1. 按行连续读取 global memory 到 shared memory
2. 在 shared memory 中完成转置访问
3. 再按行连续写回 global memory
```

示例结构：

```cpp
#define TILE_DIM 32
#define BLOCK_ROWS 8

__global__ void transposeTiled(const float* input,
                               float* output,
                               int width,
                               int height) {
    __shared__ float tile[TILE_DIM][TILE_DIM + 1];

    int x = blockIdx.x * TILE_DIM + threadIdx.x;
    int y = blockIdx.y * TILE_DIM + threadIdx.y;

    for (int j = 0; j < TILE_DIM; j += BLOCK_ROWS) {
        if (x < width && y + j < height) {
            tile[threadIdx.y + j][threadIdx.x] = input[(y + j) * width + x];
        }
    }

    __syncthreads();

    x = blockIdx.y * TILE_DIM + threadIdx.x;
    y = blockIdx.x * TILE_DIM + threadIdx.y;

    for (int j = 0; j < TILE_DIM; j += BLOCK_ROWS) {
        if (x < height && y + j < width) {
            output[(y + j) * height + x] = tile[threadIdx.x][threadIdx.y + j];
        }
    }
}
```

注意这里使用：

```cpp
__shared__ float tile[TILE_DIM][TILE_DIM + 1];
```

多加的 `+1` 是为了避免转置访问 shared memory 时产生严重 bank conflict。

## 十五、shared memory 使用原则

使用 shared memory 时，应遵循以下原则。

---

### 15.1 有数据复用时再使用

shared memory 适合缓存会被重复访问的数据。

适合：

```text
同一块数据会被 block 内多个线程多次读取。
```

不适合：

```text
每个数据只被一个线程读取一次。
```

例如，向量加法通常不需要 shared memory，因为每个线程只读取 `A[idx]` 和 `B[idx]` 一次，使用 shared memory 反而会增加额外拷贝和同步开销。

---

### 15.2 尽量让 global memory 读取合并

shared memory 优化的第一步通常是让线程以连续方式从 global memory 读取数据。

推荐：

```cpp
tile[threadIdx.x] = input[blockIdx.x * blockDim.x + threadIdx.x];
```

这样相邻线程读取相邻 global memory 地址。

---

### 15.3 使用同步保证数据有效

如果 shared memory 中的数据会被其他线程读取，则写入后需要同步：

```cpp
__syncthreads();
```

---

### 15.4 注意 bank conflict

二维 shared memory 按列访问时，应考虑 padding：

```cpp
__shared__ float tile[32][33];
```

---

### 15.5 控制 shared memory 使用量

每个 block 使用 shared memory 越多，一个 SM 上能同时驻留的 block 数可能越少，occupancy 可能下降。

因此需要平衡：

```text
shared memory 带来的访存收益
与
occupancy 下降带来的并行度损失
```

---

### 15.6 避免复杂控制流中的同步

不要把 `__syncthreads()` 放在只有部分线程会执行的分支中。

错误：

```cpp
if (threadIdx.x < 16) {
    __syncthreads();
}
```

正确：

```cpp
if (threadIdx.x < 16) {
    // work
}

__syncthreads();
```

## 十六、完整示例：shared memory 归约求和

下面给出一个可以作为模板的 shared memory 归约示例。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cmath>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void reduceSumKernel(const float* input, float* blockSums, int n) {
    extern __shared__ float s_data[];

    unsigned int tid = threadIdx.x;
    unsigned int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        s_data[tid] = input[idx];
    } else {
        s_data[tid] = 0.0f;
    }

    __syncthreads();

    for (unsigned int stride = blockDim.x / 2; stride > 0; stride >>= 1) {
        if (tid < stride) {
            s_data[tid] += s_data[tid + stride];
        }

        __syncthreads();
    }

    if (tid == 0) {
        blockSums[blockIdx.x] = s_data[0];
    }
}

int main() {
    const int n = 1 << 20;
    const size_t bytes = n * sizeof(float);

    float* h_input = static_cast<float*>(malloc(bytes));

    if (h_input == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int i = 0; i < n; ++i) {
        h_input[i] = 1.0f;
    }

    dim3 block(256);
    dim3 grid((n + block.x - 1) / block.x);

    const int numBlocks = grid.x;
    const size_t blockSumBytes = numBlocks * sizeof(float);

    float* h_blockSums = static_cast<float*>(malloc(blockSumBytes));

    if (h_blockSums == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        free(h_input);
        return EXIT_FAILURE;
    }

    float* d_input = nullptr;
    float* d_blockSums = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_input, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_blockSums, blockSumBytes));

    CUDA_CHECK(cudaMemcpy(d_input, h_input, bytes, cudaMemcpyHostToDevice));

    size_t sharedMemSize = block.x * sizeof(float);

    reduceSumKernel<<<grid, block, sharedMemSize>>>(d_input, d_blockSums, n);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_blockSums,
                          d_blockSums,
                          blockSumBytes,
                          cudaMemcpyDeviceToHost));

    double gpuSum = 0.0;
    for (int i = 0; i < numBlocks; ++i) {
        gpuSum += h_blockSums[i];
    }

    double expected = static_cast<double>(n);

    printf("GPU partial reduction sum: %.0f\n", gpuSum);
    printf("Expected sum: %.0f\n", expected);
    printf("Result: %s\n", fabs(gpuSum - expected) < 1e-5 ? "PASS" : "FAIL");

    CUDA_CHECK(cudaFree(d_input));
    CUDA_CHECK(cudaFree(d_blockSums));

    free(h_input);
    free(h_blockSums);

    return EXIT_SUCCESS;
}
```

编译命令：

```bash
nvcc -O2 shared_reduction.cu -o shared_reduction
```

运行命令：

```bash
./shared_reduction
```

预期输出：

```text
Result: PASS
```

## 十七、常见错误

## 17.1 误以为 shared memory 可以跨 block 共享

错误理解：

```text
所有 block 都可以访问同一个 shared memory 数组。
```

正确理解：

```text
shared memory 只在同一个 block 内共享，不同 block 拥有各自独立的 shared memory。
```

---

## 17.2 忘记使用 `__syncthreads()`

错误：

```cpp
tile[threadIdx.x] = input[idx];

float value = tile[(threadIdx.x + 1) % blockDim.x];
```

如果读取的是其他线程写入的数据，应加入同步：

```cpp
tile[threadIdx.x] = input[idx];

__syncthreads();

float value = tile[(threadIdx.x + 1) % blockDim.x];
```

---

## 17.3 在分支中错误使用 `__syncthreads()`

错误：

```cpp
if (threadIdx.x == 0) {
    __syncthreads();
}
```

正确：

```cpp
if (threadIdx.x == 0) {
    // work by thread 0
}

__syncthreads();
```

---

## 17.4 shared memory 数组越界

错误：

```cpp
__shared__ float tile[256];

int tid = threadIdx.x;
tile[tid + 1] = input[tid];
```

当 `tid == 255` 时，访问 `tile[256]` 越界。

正确写法需要边界判断：

```cpp
if (tid + 1 < blockDim.x) {
    tile[tid + 1] = input[tid];
}
```

---

## 17.5 动态 shared memory 大小传错

错误：

```cpp
kernel<<<grid, block>>>(...);
```

但 kernel 中使用：

```cpp
extern __shared__ float s_data[];
```

如果没有传入第三个参数，动态 shared memory 大小为 0，访问会出错。

正确：

```cpp
size_t sharedMemSize = block.x * sizeof(float);
kernel<<<grid, block, sharedMemSize>>>(...);
```

---

## 17.6 使用 shared memory 反而变慢

shared memory 并不是所有场景都能提升性能。

可能原因：

1. 数据没有复用。
2. 额外引入了 global-to-shared 拷贝开销。
3. 额外引入了 `__syncthreads()` 同步开销。
4. shared memory 使用过多降低 occupancy。
5. 发生 bank conflict。
6. global memory 原本已经是连续合并访问。

例如，向量加法通常不需要 shared memory。

---

## 17.7 二维 shared memory 列访问 bank conflict

低效写法：

```cpp
__shared__ float tile[32][32];

float value = tile[threadIdx.x][fixedCol];
```

优化写法：

```cpp
__shared__ float tile[32][33];

float value = tile[threadIdx.x][fixedCol];
```

通过每行增加一个 padding 元素，使列访问时不同线程落入不同 bank。

## 十八、回答模板

## 18.1 解释 shared memory

```text
shared memory 是 GPU 片上的高速共享存储器，作用范围是一个 thread block。同一个 block 内的所有线程可以访问同一块 shared memory，因此它常用于缓存 global memory 中会被重复访问的数据，或者用于 block 内线程协作。shared memory 的访问速度通常明显快于 global memory，但容量较小，并且使用时需要注意 __syncthreads() 同步和 bank conflict。
```

---

## 18.2 解释 shared memory 和 global memory 的区别

```text
global memory 位于 GPU 板载显存中，容量大，所有线程都可以访问，但访问延迟高；shared memory 位于 GPU 片上，容量小，访问速度快，但只能被同一个 block 内的线程共享。global memory 适合保存大规模输入输出数组，而 shared memory 适合缓存 block 内会重复访问的数据，减少 global memory 访问次数。
```

---

## 18.3 解释什么时候使用 shared memory

```text
当同一块数据会被同一个 block 内多个线程重复访问，或者线程之间需要共享中间结果时，适合使用 shared memory。例如矩阵乘法分块、图像滤波邻域缓存、并行归约和矩阵转置都适合使用 shared memory。如果每个数据只被一个线程读取一次，例如简单向量加法，使用 shared memory 往往没有收益，甚至会增加拷贝和同步开销。
```

---

## 18.4 解释 `__syncthreads()`

```text
__syncthreads() 用于同步同一个 block 内的所有线程。它通常用于 shared memory 编程中，确保所有线程都完成写入后，其他线程再读取这些数据。需要注意的是，__syncthreads() 只能同步同一个 block 内线程，不能同步不同 block，并且同一个 block 内所有线程必须以一致的控制流执行到该同步点，否则可能导致死锁。
```

---

## 18.5 解释 bank conflict

```text
shared memory 被划分为多个 bank。同一个 warp 中，如果多个线程同时访问不同 bank，访问可以并行完成；如果多个线程访问同一个 bank 中的不同地址，就会产生 bank conflict，访问可能被串行化，从而降低性能。二维 shared memory 按列访问时容易发生 bank conflict，常见优化方法是在每行末尾增加一个 padding 元素，例如使用 tile[32][33] 代替 tile[32][32]。
```

---

## 18.6 解释 shared memory 矩阵乘法优化

```text
在基础矩阵乘法中，每个线程直接从 global memory 读取 A 和 B 的数据，很多数据会被不同线程重复读取。shared memory 优化的思想是将 A 和 B 的小块 tile 先加载到 shared memory 中，同一个 block 内的线程重复使用这些 tile 进行计算。这样可以减少 global memory 访问次数，提高数据复用率。每轮 tile 加载后需要使用 __syncthreads() 保证数据完整，再进行乘加计算。
```

## 二十、小结

shared memory 是 CUDA 中非常重要的片上高速存储器。它由同一个 block 内的线程共享，适合缓存可复用数据和实现线程协作。与 global memory 相比，shared memory 容量较小，但访问速度更快，因此常用于减少重复 global memory 访问。

使用 shared memory 时，最重要的是判断数据是否存在复用。如果数据只被读取一次，使用 shared memory 通常没有明显收益；如果数据会被同一个 block 内多个线程重复使用，例如矩阵乘法中的 tile、图像滤波中的邻域数据、归约中的中间结果，则 shared memory 能显著减少 global memory 访问开销。

shared memory 编程还必须注意同步和 bank conflict。多个线程写入 shared memory 后，如果其他线程要读取这些数据，通常需要使用 `__syncthreads()`。二维 shared memory 按列访问时可能产生 bank conflict，常见优化方法是在每行增加一个 padding 元素，例如 `tile[32][33]`。

本节的核心结论是：shared memory 不是自动加速工具，而是一种需要配合数据复用、线程协作和合理访问模式使用的片上缓存。只有在访问模式设计正确时，它才能真正提升 CUDA 程序性能。
