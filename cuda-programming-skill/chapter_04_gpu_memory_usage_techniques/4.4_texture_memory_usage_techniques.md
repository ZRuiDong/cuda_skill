# 4.4 纹理存储器的使用技巧

## 一、本节作用

本节用于整理 CUDA 纹理存储器 texture memory 的基本概念、使用方法和适用场景。

纹理存储器最初来源于图形学中的纹理访问机制，后来也被用于通用计算。它是一种只读存储器访问方式，带有专门的缓存机制，适合访问具有空间局部性的数据，例如图像、二维矩阵、体数据和采样数据。

在教材中，纹理存储器主要通过 texture reference 方式使用，也就是先声明纹理参考，再将线性存储器或 CUDA 数组绑定到纹理参考，然后在 kernel 中使用纹理拾取函数读取数据。现代 CUDA 中，texture reference 相关接口已经不再是推荐写法，更推荐使用 texture object。因此，本节同时整理两部分内容：

1. 教材风格的 texture reference 写法。
2. 现代 CUDA 中的 texture object 写法。

本节完成后，应能够回答以下问题：

1. texture memory 是什么。
2. texture memory 与 global memory 有什么区别。
3. texture memory 与 constant memory 有什么区别。
4. 什么数据适合使用 texture memory。
5. 线性存储器和 CUDA 数组如何绑定到纹理。
6. `tex1Dfetch`、`tex1D`、`tex2D`、`tex3D` 有什么区别。
7. 纹理坐标 normalized coordinates 是什么。
8. address mode 和 filter mode 有什么作用。
9. 传统 texture reference 和现代 texture object 有什么区别。
10. 图像处理任务中如何使用纹理存储器。

---

## 二、texture memory 的基本特点

texture memory，即纹理存储器，是一种 device 端只读访问机制。它并不是一种完全独立于显存之外的大容量存储器，而是通过纹理缓存访问 global memory 或 CUDA array 中的数据。

texture memory 的主要特点如下：

1. device 端只读。
2. 带有纹理缓存。
3. 适合具有空间局部性的数据访问。
4. 对随机访问和二维局部访问比普通 global memory 更友好。
5. 支持一维、二维、三维纹理读取。
6. 可支持坐标寻址。
7. 可支持边界寻址模式。
8. 可支持硬件插值过滤。
9. 可支持部分类型转换。
10. 常用于图像处理、体绘制和采样类算法。

纹理存储器的核心优势是：

```text
当线程访问的数据在空间上接近，但不一定满足 global memory 合并访问要求时，texture cache 可能提高读取效率。
```

---

## 三、texture memory 适合的访问模式

texture memory 适合以下访问模式：

1. 只读访问。
2. 二维或三维空间数据访问。
3. 具有空间局部性的访问。
4. 图像像素邻域访问。
5. 体数据采样。
6. 不规则但局部性较强的读取。
7. 需要边界处理的采样访问。
8. 需要插值的采样访问。

典型应用包括：

1. 图像滤波。
2. 图像缩放。
3. 图像旋转。
4. 纹理采样。
5. 体绘制。
6. 光线投射。
7. 地震数据三维可视化。
8. 医学图像重建。
9. 有限差分中的邻域读取。
10. 查找表类只读访问。

不适合 texture memory 的场景：

1. 数据需要在 kernel 中写入。
2. 访问模式已经是完全连续合并访问。
3. 数据没有明显空间局部性。
4. 简单向量加法等一次性线性访问任务。
5. 需要频繁修改的数据。

---

## 四、texture memory 与 global memory 的区别

texture memory 和 global memory 都可以访问 GPU 显存中的数据，但访问机制不同。

| 对比项 | global memory | texture memory |
|---|---|---|
| 访问权限 | device 可读写 | device 只读 |
| 缓存机制 | 依赖普通全局缓存层次 | 使用纹理缓存 |
| 访问模式 | 连续合并访问最优 | 空间局部访问较友好 |
| 坐标模式 | 普通线性地址 | 可使用纹理坐标 |
| 插值能力 | 无硬件插值 | 可支持线性插值 |
| 边界处理 | 手动判断 | 可使用 address mode |
| 典型用途 | 输入输出数组、大规模数据 | 图像、体数据、采样数据 |

简单判断：

```text
如果线程连续访问连续地址，global memory 已经很高效。
如果线程访问二维或三维邻域数据，且数据只读，可以考虑 texture memory。
```

---

## 五、texture memory 与 constant memory 的区别

texture memory 和 constant memory 都是 device 端只读访问机制，并且都带有缓存，但适用场景不同。

| 对比项 | constant memory | texture memory |
|---|---|---|
| 典型数据规模 | 小 | 可较大 |
| 访问模式 | 多个线程读取同一地址较高效 | 空间局部访问较高效 |
| 典型用途 | 卷积核、参数、小查找表 | 图像、矩阵、体数据 |
| 坐标访问 | 无纹理坐标 | 支持纹理坐标 |
| 插值 | 不支持 | 可支持 |
| 边界模式 | 通常手动处理 | 可通过 address mode 处理 |

判断方法：

```text
小规模只读参数：优先 constant memory
图像或体数据只读采样：考虑 texture memory
```

例如：

1. 3×3 高斯滤波模板适合 constant memory。
2. 输入图像本身可以考虑 texture memory。
3. 体绘制中的三维体数据适合 texture memory。
4. 多个线程读取同一个常量参数时，constant memory 更合适。

---

## 六、绑定到纹理的数据类型

教材中说明，CUDA 中与纹理绑定的数据主要有两类：

1. 线性存储器 linear memory。
2. CUDA 数组 CUDA array。

---

## 6.1 线性存储器

线性存储器通常通过以下函数分配：

```cpp
cudaMalloc
cudaMallocPitch
cudaMalloc3D
```

其中，`cudaMalloc` 分配普通一维线性 global memory；`cudaMallocPitch` 分配带 pitch 的二维线性空间；`cudaMalloc3D` 可用于三维线性空间。

线性存储器绑定到纹理后，通常使用：

```cpp
tex1Dfetch()
```

进行访问。

`tex1Dfetch()` 使用整数坐标，适合一维线性数组读取。

---

## 6.2 CUDA 数组

CUDA array 是一种专门用于纹理或表面访问的数据结构，适合一维、二维或三维纹理访问。

CUDA array 通常通过以下函数分配：

```cpp
cudaMallocArray
cudaMalloc3DArray
```

数据复制可以使用：

```cpp
cudaMemcpyToArray
cudaMemcpy2DToArray
cudaMemcpy3D
```

CUDA array 绑定到纹理后，通常使用：

```cpp
tex1D()
tex2D()
tex3D()
```

进行访问。

这些函数使用浮点坐标，适合一维、二维或三维空间采样。

---

## 七、纹理参考 texture reference

## 7.1 texture reference 的概念

texture reference，即纹理参考，是教材中使用纹理存储器的核心对象。它用于描述 kernel 中如何读取某一段纹理数据。

使用 texture reference 时，需要经过以下步骤：

```text
1. 声明 texture reference
2. 在 host 端分配或获得数据
3. 将数据绑定到 texture reference
4. 在 kernel 中通过纹理拾取函数读取
5. kernel 执行结束后解除绑定
```

texture reference 是 host 端绑定数据和 device 端读取数据之间的桥梁。

---

## 7.2 texture reference 声明格式

教材风格的纹理参考声明通常写成：

```cpp
texture<Type, Dim, ReadMode> texRef;
```

其中：

- `Type` 表示纹理元素类型。
- `Dim` 表示纹理维度，可以是 1、2 或 3。
- `ReadMode` 表示读取模式。

示例：

```cpp
texture<float, 1, cudaReadModeElementType> tex1DRef;
```

二维纹理：

```cpp
texture<float, 2, cudaReadModeElementType> tex2DRef;
```

整数纹理：

```cpp
texture<int, 1, cudaReadModeElementType> texIntRef;
```

---

## 7.3 `cudaReadModeElementType`

`cudaReadModeElementType` 表示纹理读取时返回原始元素类型。

例如：

```cpp
texture<float, 1, cudaReadModeElementType> texRef;
```

如果绑定的数据是 `float`，那么读取返回的也是 `float`。

---

## 7.4 `cudaReadModeNormalizedFloat`

`cudaReadModeNormalizedFloat` 表示读取整数纹理数据时，将结果转换为归一化浮点值。

例如，对于 8 bit 或 16 bit 整数数据，读取时可以转换到类似：

```text
[0.0, 1.0]
```

或：

```text
[-1.0, 1.0]
```

的范围。

这种模式在图像处理中较常见，例如读取灰度图像或颜色通道时，把整数像素值转换为浮点值参与计算。

---

## 八、运行时纹理参考属性

texture reference 可以设置一些运行时属性，影响纹理坐标解释、边界处理和插值行为。

常见属性包括：

1. `normalized`
2. `addressMode`
3. `filterMode`
4. `readMode`

---

## 8.1 normalized 坐标

`normalized` 表示是否使用规范化纹理坐标。

如果不使用规范化坐标：

```text
坐标范围通常是 [0, N)
```

例如，一个大小为 64×32 的纹理：

```text
x 坐标范围：[0, 64)
y 坐标范围：[0, 32)
```

如果使用规范化坐标：

```text
坐标范围通常是 [0.0, 1.0)
```

同一个 64×32 纹理：

```text
x 坐标范围：[0.0, 1.0)
y 坐标范围：[0.0, 1.0)
```

示例设置：

```cpp
texRef.normalized = false;
```

或：

```cpp
texRef.normalized = true;
```

初学阶段通常使用非规范化坐标，这样更接近数组下标。

---

## 8.2 addressMode 寻址模式

`addressMode` 定义纹理坐标超出范围时如何处理。

常见模式包括：

```cpp
cudaAddressModeClamp
cudaAddressModeWrap
```

---

### `cudaAddressModeClamp`

clamp 表示夹紧。

如果坐标小于最小范围，就使用最小边界值；如果坐标超过最大范围，就使用最大边界值。

例如一维纹理长度为 `N`：

```text
x < 0      -> 0
x >= N     -> N - 1
```

这种模式适合图像边界处理。

---

### `cudaAddressModeWrap`

wrap 表示循环寻址。

例如规范化坐标中：

```text
1.25  -> 0.25
-1.25 -> 0.75
```

wrap 模式适合周期信号、周期纹理或循环边界条件。

需要注意，wrap 通常与规范化坐标配合使用。

---

## 8.3 filterMode 过滤模式

`filterMode` 定义纹理读取时是否进行插值。

常见模式包括：

```cpp
cudaFilterModePoint
cudaFilterModeLinear
```

---

### `cudaFilterModePoint`

point 模式表示点采样，通常返回最接近坐标位置的 texel。

适合：

1. 不需要插值的图像访问。
2. 精确读取数组元素。
3. 离散数据读取。

---

### `cudaFilterModeLinear`

linear 模式表示线性插值。

对于一维纹理，执行线性插值；对于二维纹理，执行双线性插值；对于三维纹理，可以执行三线性插值。

适合：

1. 图像缩放。
2. 图像旋转。
3. 体数据采样。
4. 需要平滑采样的任务。

需要注意：线性过滤通常要求纹理配置为返回浮点数据。

---

## 8.4 readMode 读取模式

readMode 决定纹理拾取函数返回的数据类型方式。

常见模式：

```cpp
cudaReadModeElementType
cudaReadModeNormalizedFloat
```

其中：

- `cudaReadModeElementType` 返回原始类型。
- `cudaReadModeNormalizedFloat` 将整数类型归一化为浮点值。

---

## 九、纹理存储器的使用流程

教材中纹理存储器的使用流程可以概括为三部分：

1. host 端建立纹理。
2. device 端纹理拾取。
3. host 端删除纹理。

---

## 9.1 host 端建立纹理

host 端通常需要完成：

```text
1. 分配或获得存储空间
2. 声明 texture reference
3. 设置 texture reference 属性
4. 将数据绑定到 texture reference
```

线性存储器绑定：

```cpp
cudaBindTexture()
```

CUDA 数组绑定：

```cpp
cudaBindTextureToArray()
```

---

## 9.2 device 端纹理拾取

kernel 中使用纹理拾取函数读取数据。

线性存储器绑定时常用：

```cpp
tex1Dfetch()
```

CUDA 数组绑定时常用：

```cpp
tex1D()
tex2D()
tex3D()
```

---

## 9.3 host 端解除绑定

使用完成后，需要解除绑定：

```cpp
cudaUnbindTexture()
```

同时释放相应的 device memory 或 CUDA array。

---

## 十、texture fetch 函数

## 10.1 `tex1Dfetch`

`tex1Dfetch` 用于读取绑定到线性存储器的一维纹理。

特点：

1. 使用整数坐标。
2. 适合一维数组。
3. 常与 `cudaBindTexture` 搭配。
4. 不使用 normalized 坐标。

示例：

```cpp
float value = tex1Dfetch(texRef, idx);
```

---

## 10.2 `tex1D`

`tex1D` 用于读取一维 CUDA array 纹理。

特点：

1. 使用浮点坐标。
2. 可配合 normalized 坐标。
3. 可配合过滤模式。

示例：

```cpp
float value = tex1D(texRef, x);
```

---

## 10.3 `tex2D`

`tex2D` 用于读取二维 CUDA array 纹理。

特点：

1. 使用二维浮点坐标。
2. 适合图像和二维矩阵。
3. 可配合 address mode。
4. 可配合 filter mode。
5. 常用于图像处理和采样。

示例：

```cpp
float value = tex2D(texRef, x, y);
```

---

## 10.4 `tex3D`

`tex3D` 用于读取三维 CUDA array 纹理。

特点：

1. 使用三维浮点坐标。
2. 适合体数据。
3. 常用于体绘制和三维采样。
4. 可配合三线性插值。

示例：

```cpp
float value = tex3D(texRef, x, y, z);
```

## 十一、教材风格示例：线性存储器绑定到 texture reference

下面示例展示一维数组绑定到纹理参考，并在 kernel 中使用 `tex1Dfetch` 读取。

注意：texture reference 是教材风格写法，在现代 CUDA 中可能不再推荐用于新项目。保留该示例主要用于理解教材内容。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cmath>
#include <cuda_runtime.h>

texture<float, 1, cudaReadModeElementType> texRef;

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void textureReadKernel(float* output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        float value = tex1Dfetch(texRef, idx);
        output[idx] = value * 2.0f;
    }
}

int main() {
    const int n = 1 << 20;
    const size_t bytes = n * sizeof(float);

    float* h_input = static_cast<float*>(malloc(bytes));
    float* h_output = static_cast<float*>(malloc(bytes));

    if (h_input == nullptr || h_output == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int i = 0; i < n; ++i) {
        h_input[i] = static_cast<float>(i);
    }

    float* d_input = nullptr;
    float* d_output = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_input, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_output, bytes));

    CUDA_CHECK(cudaMemcpy(d_input, h_input, bytes, cudaMemcpyHostToDevice));

    CUDA_CHECK(cudaBindTexture(nullptr, texRef, d_input, bytes));

    dim3 block(256);
    dim3 grid((n + block.x - 1) / block.x);

    textureReadKernel<<<grid, block>>>(d_output, n);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_output, d_output, bytes, cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < n; ++i) {
        float expected = h_input[i] * 2.0f;

        if (fabs(h_output[i] - expected) > 1e-5f) {
            correct = false;
            fprintf(stderr, "Mismatch at %d: got %f, expected %f\n",
                    i, h_output[i], expected);
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaUnbindTexture(texRef));
    CUDA_CHECK(cudaFree(d_input));
    CUDA_CHECK(cudaFree(d_output));

    free(h_input);
    free(h_output);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

编译命令：

```bash
nvcc -O2 texture_linear_reference.cu -o texture_linear_reference
```

## 十二、教材风格示例：二维 CUDA 数组绑定到 texture reference

二维纹理更能体现 texture memory 的特点。下面示例展示二维图像通过 CUDA array 绑定到纹理参考，并使用 `tex2D` 读取。

注意：这是教材风格写法，用于理解旧式 texture reference 流程。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cmath>
#include <cuda_runtime.h>

texture<float, 2, cudaReadModeElementType> tex2DRef;

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void copyTexture2DKernel(float* output, int width, int height) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;

    if (x < width && y < height) {
        float value = tex2D(tex2DRef, static_cast<float>(x), static_cast<float>(y));
        output[y * width + x] = value;
    }
}

int main() {
    const int width = 512;
    const int height = 512;
    const size_t bytes = width * height * sizeof(float);

    float* h_input = static_cast<float*>(malloc(bytes));
    float* h_output = static_cast<float*>(malloc(bytes));

    if (h_input == nullptr || h_output == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int y = 0; y < height; ++y) {
        for (int x = 0; x < width; ++x) {
            h_input[y * width + x] = static_cast<float>(y * width + x);
        }
    }

    cudaChannelFormatDesc channelDesc = cudaCreateChannelDesc<float>();

    cudaArray* cuArray = nullptr;
    CUDA_CHECK(cudaMallocArray(&cuArray, &channelDesc, width, height));

    CUDA_CHECK(cudaMemcpy2DToArray(cuArray,
                                   0,
                                   0,
                                   h_input,
                                   width * sizeof(float),
                                   width * sizeof(float),
                                   height,
                                   cudaMemcpyHostToDevice));

    tex2DRef.normalized = false;
    tex2DRef.addressMode[0] = cudaAddressModeClamp;
    tex2DRef.addressMode[1] = cudaAddressModeClamp;
    tex2DRef.filterMode = cudaFilterModePoint;

    CUDA_CHECK(cudaBindTextureToArray(tex2DRef, cuArray, channelDesc));

    float* d_output = nullptr;
    CUDA_CHECK(cudaMalloc((void**)&d_output, bytes));

    dim3 block(16, 16);
    dim3 grid((width + block.x - 1) / block.x,
              (height + block.y - 1) / block.y);

    copyTexture2DKernel<<<grid, block>>>(d_output, width, height);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_output, d_output, bytes, cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < width * height; ++i) {
        if (fabs(h_output[i] - h_input[i]) > 1e-5f) {
            correct = false;
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaUnbindTexture(tex2DRef));
    CUDA_CHECK(cudaFreeArray(cuArray));
    CUDA_CHECK(cudaFree(d_output));

    free(h_input);
    free(h_output);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

## 十三、现代 CUDA：texture object

## 13.1 为什么使用 texture object

texture reference 是教材中常见的旧式写法。现代 CUDA 中，更推荐使用 texture object。

texture object 的优点包括：

1. 不依赖全局 texture reference 声明。
2. 可以作为 kernel 参数传递。
3. 更适合模块化程序设计。
4. 可以在运行时创建多个纹理对象。
5. 更符合现代 CUDA Runtime API 风格。
6. 更适合多纹理、多对象、多模块场景。

texture object 的基本类型是：

```cpp
cudaTextureObject_t
```

kernel 中可以将它作为参数传入：

```cpp
__global__ void kernel(cudaTextureObject_t texObj, float* output) {
    float value = tex2D<float>(texObj, x, y);
}
```

---

## 13.2 texture object 使用流程

现代 texture object 使用流程如下：

```text
1. 准备数据
2. 创建 CUDA array 或 linear resource
3. 填写 cudaResourceDesc
4. 填写 cudaTextureDesc
5. 调用 cudaCreateTextureObject
6. kernel 中使用 tex1D / tex2D / tex3D 读取
7. 调用 cudaDestroyTextureObject
8. 释放底层资源
```

---

## 十四、现代 CUDA 示例：二维 texture object

下面是一个现代 CUDA 风格的二维 texture object 示例。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cstring>
#include <cmath>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void textureObject2DKernel(cudaTextureObject_t texObj,
                                      float* output,
                                      int width,
                                      int height) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;

    if (x < width && y < height) {
        float value = tex2D<float>(texObj,
                                   static_cast<float>(x),
                                   static_cast<float>(y));

        output[y * width + x] = value;
    }
}

int main() {
    const int width = 512;
    const int height = 512;
    const size_t bytes = width * height * sizeof(float);

    float* h_input = static_cast<float*>(malloc(bytes));
    float* h_output = static_cast<float*>(malloc(bytes));

    if (h_input == nullptr || h_output == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int y = 0; y < height; ++y) {
        for (int x = 0; x < width; ++x) {
            h_input[y * width + x] = static_cast<float>(y * width + x);
        }
    }

    cudaChannelFormatDesc channelDesc = cudaCreateChannelDesc<float>();

    cudaArray_t cuArray = nullptr;
    CUDA_CHECK(cudaMallocArray(&cuArray, &channelDesc, width, height));

    CUDA_CHECK(cudaMemcpy2DToArray(cuArray,
                                   0,
                                   0,
                                   h_input,
                                   width * sizeof(float),
                                   width * sizeof(float),
                                   height,
                                   cudaMemcpyHostToDevice));

    cudaResourceDesc resDesc;
    std::memset(&resDesc, 0, sizeof(resDesc));
    resDesc.resType = cudaResourceTypeArray;
    resDesc.res.array.array = cuArray;

    cudaTextureDesc texDesc;
    std::memset(&texDesc, 0, sizeof(texDesc));
    texDesc.addressMode[0] = cudaAddressModeClamp;
    texDesc.addressMode[1] = cudaAddressModeClamp;
    texDesc.filterMode = cudaFilterModePoint;
    texDesc.readMode = cudaReadModeElementType;
    texDesc.normalizedCoords = 0;

    cudaTextureObject_t texObj = 0;
    CUDA_CHECK(cudaCreateTextureObject(&texObj, &resDesc, &texDesc, nullptr));

    float* d_output = nullptr;
    CUDA_CHECK(cudaMalloc((void**)&d_output, bytes));

    dim3 block(16, 16);
    dim3 grid((width + block.x - 1) / block.x,
              (height + block.y - 1) / block.y);

    textureObject2DKernel<<<grid, block>>>(texObj, d_output, width, height);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_output, d_output, bytes, cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < width * height; ++i) {
        if (fabs(h_output[i] - h_input[i]) > 1e-5f) {
            correct = false;
            fprintf(stderr, "Mismatch at %d: got %f, expected %f\n",
                    i, h_output[i], h_input[i]);
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaDestroyTextureObject(texObj));
    CUDA_CHECK(cudaFreeArray(cuArray));
    CUDA_CHECK(cudaFree(d_output));

    free(h_input);
    free(h_output);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

编译命令：

```bash
nvcc -O2 -std=c++17 texture_object_2d.cu -o texture_object_2d
```

运行命令：

```bash

```

预期输出：

```text

```

## 十五、现代 CUDA 示例：线性内存 texture object

texture object 也可以绑定线性内存。下面示例展示一维数组的线性资源绑定。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cstring>
#include <cmath>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void linearTextureObjectKernel(cudaTextureObject_t texObj,
                                          float* output,
                                          int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        float value = tex1Dfetch<float>(texObj, idx);
        output[idx] = value * 2.0f;
    }
}

int main() {
    const int n = 1 << 20;
    const size_t bytes = n * sizeof(float);

    float* h_input = static_cast<float*>(malloc(bytes));
    float* h_output = static_cast<float*>(malloc(bytes));

    if (h_input == nullptr || h_output == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int i = 0; i < n; ++i) {
        h_input[i] = static_cast<float>(i);
    }

    float* d_input = nullptr;
    float* d_output = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_input, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_output, bytes));

    CUDA_CHECK(cudaMemcpy(d_input, h_input, bytes, cudaMemcpyHostToDevice));

    cudaResourceDesc resDesc;
    std::memset(&resDesc, 0, sizeof(resDesc));
    resDesc.resType = cudaResourceTypeLinear;
    resDesc.res.linear.devPtr = d_input;
    resDesc.res.linear.desc = cudaCreateChannelDesc<float>();
    resDesc.res.linear.sizeInBytes = bytes;

    cudaTextureDesc texDesc;
    std::memset(&texDesc, 0, sizeof(texDesc));
    texDesc.readMode = cudaReadModeElementType;
    texDesc.normalizedCoords = 0;

    cudaTextureObject_t texObj = 0;
    CUDA_CHECK(cudaCreateTextureObject(&texObj, &resDesc, &texDesc, nullptr));

    dim3 block(256);
    dim3 grid((n + block.x - 1) / block.x);

    linearTextureObjectKernel<<<grid, block>>>(texObj, d_output, n);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_output, d_output, bytes, cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < n; ++i) {
        float expected = h_input[i] * 2.0f;

        if (fabs(h_output[i] - expected) > 1e-5f) {
            correct = false;
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaDestroyTextureObject(texObj));
    CUDA_CHECK(cudaFree(d_input));
    CUDA_CHECK(cudaFree(d_output));

    free(h_input);
    free(h_output);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

编译命令：

```bash
nvcc -O2 -std=c++17 texture_object_linear.cu -o texture_object_linear
```

## 十六、图像处理中使用 texture memory 的思路

图像处理是 texture memory 的典型应用场景。

以图像缩放或图像旋转为例，输出图像中的每个像素通常需要从输入图像的非整数位置采样。如果使用普通 global memory，需要手动处理：

1. 坐标边界。
2. 邻近像素读取。
3. 双线性插值。
4. 像素类型转换。

而使用 texture memory 时，可以借助：

```text
addressMode
filterMode
normalized coordinates
```

简化采样过程。

例如启用双线性插值：

```cpp
texDesc.filterMode = cudaFilterModeLinear;
```

使用 clamp 边界：

```cpp
texDesc.addressMode[0] = cudaAddressModeClamp;
texDesc.addressMode[1] = cudaAddressModeClamp;
```

这样在 kernel 中采样时：

```cpp
float value = tex2D<float>(texObj, x, y);
```

即可获得纹理缓存和硬件插值带来的便利。

---

## 十七、使用 texture memory 的基本判断

可以按以下问题判断是否适合使用 texture memory：

1. 数据是否只读？
2. 数据是否是一维、二维或三维空间数据？
3. 线程访问是否具有空间局部性？
4. 访问是否不是简单连续合并访问？
5. 是否需要坐标边界处理？
6. 是否需要插值？
7. 是否用于图像、体数据或采样？
8. 数据是否会在 kernel 中被修改？

如果答案是：

```text
只读 + 空间局部性 + 采样访问
```

则可以考虑 texture memory。

如果是：

```text
线性数组 + 连续线程访问连续地址 + 不需要插值
```

通常直接使用 global memory 即可。

## 十八、常见错误

## 18.1 在 kernel 中写 texture memory

错误理解：

```text
texture memory 可以像 global memory 一样读写。
```

正确理解：

```text
texture memory 在 device 端用于只读访问。
```

如果需要写入结果，应写入 global memory 或 surface memory，而不是 texture memory。

---

## 18.2 忘记绑定纹理

教材风格 texture reference 中，如果只声明：

```cpp
texture<float, 1, cudaReadModeElementType> texRef;
```

但没有调用：

```cpp
cudaBindTexture()
```

kernel 中读取纹理会出错或得到无效结果。

---

## 18.3 忘记解除绑定或释放资源

使用 texture reference 后，应解除绑定：

```cpp
cudaUnbindTexture(texRef);
```

使用 CUDA array 后，应释放：

```cpp
cudaFreeArray(cuArray);
```

使用 texture object 后，应销毁对象：

```cpp
cudaDestroyTextureObject(texObj);
```

---

## 18.4 坐标类型使用错误

`tex1Dfetch` 使用整数坐标：

```cpp
float v = tex1Dfetch(texRef, idx);
```

`tex2D` 使用浮点坐标：

```cpp
float v = tex2D(texRef, x, y);
```

如果使用 normalized coordinates，则坐标范围不再是 `[0, width)` 和 `[0, height)`，而通常是 `[0.0, 1.0)`。

---

## 18.5 normalized 坐标理解错误

错误理解：

```text
normalized = true 时，x 仍然传像素下标。
```

正确理解：

```text
normalized = true 时，坐标通常应归一化到 [0.0, 1.0)。
```

例如图像宽度为 `width`，像素坐标 `x` 可转换为：

```cpp
float u = static_cast<float>(x) / static_cast<float>(width);
```

---

## 18.6 filterMode 使用错误

如果启用：

```cpp
cudaFilterModeLinear
```

但数据类型或读取模式不适合线性插值，可能无法得到预期效果。

线性过滤更适合返回浮点数据的纹理采样场景。

---

## 18.7 把 texture memory 当作通用优化手段

texture memory 并不是所有场景都能提升性能。

对于简单向量加法：

```cpp
C[idx] = A[idx] + B[idx];
```

如果 global memory 已经是连续合并访问，使用 texture memory 通常没有明显优势，还会增加绑定和代码复杂度。

---

## 18.8 新项目继续使用旧式 texture reference

教材使用 texture reference 是因为它对应早期 CUDA 编程方式。现代 CUDA 项目中，更推荐使用 texture object。除非用户明确要求按照教材旧式写法，否则新代码优先给出 texture object 版本。

## 十九、回答模板

## 19.1 解释 texture memory

```text
texture memory 是 CUDA 中用于只读访问的一种纹理存储机制。它通常绑定到 global memory 或 CUDA array 上，在 kernel 中通过 tex1Dfetch、tex1D、tex2D 或 tex3D 等纹理拾取函数读取。texture memory 带有纹理缓存，适合图像、二维数组、三维体数据等具有空间局部性的访问场景，并且可以支持坐标寻址、边界处理、硬件插值和部分类型转换。
```

---

## 19.2 解释 texture memory 与 global memory 的区别

```text
global memory 是通用的大容量 device memory，可以读写，但访问效率高度依赖连续合并访问。texture memory 是只读访问机制，底层数据也可以来自 global memory 或 CUDA array，但读取时经过纹理缓存，适合空间局部性较强、访问模式不完全连续的图像或采样类任务。如果线程只是连续访问连续数组，global memory 通常已经足够；如果访问二维邻域、图像采样或体数据采样，可以考虑 texture memory。
```

---

## 19.3 解释 texture reference 的使用流程

```text
教材中的 texture reference 使用流程通常包括三步。第一步，在 host 端分配或获得数据，并声明 texture reference，然后使用 cudaBindTexture 或 cudaBindTextureToArray 将线性存储器或 CUDA array 绑定到纹理参考。第二步，在 device 端 kernel 中使用 tex1Dfetch、tex1D、tex2D 或 tex3D 读取纹理数据。第三步，在 host 端调用 cudaUnbindTexture 解除绑定，并释放相应资源。
```

---

## 19.4 解释 `tex1Dfetch` 和 `tex2D` 的区别

```text
tex1Dfetch 通常用于读取绑定到线性存储器的一维纹理，使用整数坐标，适合普通一维数组访问。tex2D 通常用于读取绑定到 CUDA array 的二维纹理，使用浮点坐标，适合图像或二维数据访问，并且可以配合 normalized coordinates、addressMode 和 filterMode 实现边界处理和插值采样。
```

---

## 19.5 解释 texture object

```text
texture object 是现代 CUDA 中更推荐的纹理访问方式。它通过 cudaResourceDesc 和 cudaTextureDesc 描述底层资源和采样属性，再调用 cudaCreateTextureObject 创建 cudaTextureObject_t。与旧式 texture reference 相比，texture object 可以作为 kernel 参数传递，更适合模块化和多纹理对象场景。使用完成后需要调用 cudaDestroyTextureObject 销毁。
```

---

## 19.6 解释什么时候使用 texture memory

```text
当数据只读，并且访问模式具有空间局部性，例如图像采样、二维滤波、图像缩放、图像旋转、体数据采样或体绘制时，可以考虑使用 texture memory。如果数据需要在 kernel 中写入，或者线程访问已经是连续合并访问，texture memory 通常不是首选。
```

## 二十一、小结

texture memory 是 CUDA 中面向只读数据访问的一种重要机制。它通过纹理缓存提高具有空间局部性的数据访问效率，常用于图像、二维矩阵、三维体数据和采样类计算。与 global memory 相比，texture memory 更适合访问模式不完全连续但局部性较强的数据；与 constant memory 相比，texture memory 更适合较大规模的空间数据，而 constant memory 更适合小规模只读参数。

教材中的纹理存储器使用方式主要基于 texture reference。其基本流程是：先声明纹理参考，将线性存储器或 CUDA 数组绑定到纹理参考，然后在 kernel 中通过 `tex1Dfetch`、`tex1D`、`tex2D` 或 `tex3D` 读取，最后在 host 端解除绑定。

现代 CUDA 中，更推荐使用 texture object。texture object 使用 `cudaResourceDesc` 和 `cudaTextureDesc` 描述资源和采样方式，通过 `cudaCreateTextureObject` 创建，并作为参数传入 kernel。它比旧式 texture reference 更灵活，也更适合现代 CUDA 工程。

本节的核心结论是：texture memory 不是通用替代 global memory 的工具，而是适合只读、空间局部、采样访问的一类特殊访存优化手段。使用时应结合数据访问模式判断是否值得引入。
