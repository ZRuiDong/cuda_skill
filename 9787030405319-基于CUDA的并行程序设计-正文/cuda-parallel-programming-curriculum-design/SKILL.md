# CUDA并行编程课程体系设计

## 触发条件
当用户提出以下需求之一时激活本技能：
- “如何系统学习CUDA？”
- “设计一门CUDA并行编程课程”
- “组织一本CUDA技术书籍的内容结构”
- “为遥感/医学成像等领域定制CUDA学习计划”

## 执行步骤

### 1. 确定学习者背景
根据变量 `learner_background` 选择路径：
- **初学者**：必须从基础理论模块（第1–5章）开始，按顺序学习。
- **领域专家**：可直接跳转至对应应用实例章节（第6–9章）。
- **教育者**：采用“前5周理论 + 后4周项目实践”的教学安排。

### 2. 识别目标应用领域
根据 `target_application` 匹配应用章节：
- **遥感影像处理** → 第6章
- **体绘制与可视化** → 第7章
- **生物细胞图像诊断** → 第8章
- **大规模数据处理** → 第9章

### 3. 确认主要编程语言
根据 `programming_language` 调整重点：
- **C++**：强调内核函数、内存管理与性能优化。
- **MATLAB**：聚焦GPU Coder集成与图像处理函数加速。
- **OpenGL_C**：结合光线投射与GPU纹理内存优化。

### 4. 生成个性化学习路线图
输出包含以下要素的结构化建议：
- 必读章节列表
- 可跳过章节（针对领域专家）
- 推荐实验/项目（按应用领域）
- 硬件与环境准备提示（需GPU支持）

## 注意事项
- 本技能不适用于纯理论并行算法研究。
- 所有路径均需配套GPU硬件环境。
- 跨语言实现时，需强调同一优化原理在不同语言中的差异。
