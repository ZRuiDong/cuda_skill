# 并行算法时间复杂度与耗费分析

## 何时使用
- 需要评估并行算法的理论性能上限
- 比较不同并行策略的效率
- 判断并行化是否带来成本效益（即总耗费是否接近串行时间复杂度）
- 进行并行加速比或可扩展性初步分析

## 执行步骤
1. **确认前提条件**：
   - 已知输入规模 `n`
   - 已知并行处理器数量 `P`
   - 算法执行步骤明确且可在PRAM模型下描述

2. **计算时间复杂度**：
   - 时间复杂度 = 从第一台处理器开始执行到最后一台完成所需的最大并行步数
   - 常见模式如树形归约的时间复杂度为 O(log₂n)

3. **计算并行耗费（Work）**：
   - 耗费 = 时间复杂度 × P

4. **评估性能意义**：
   - 若耗费 ≈ 串行时间复杂度（如 O(n)），则为高效并行
   - 若耗费显著大于串行复杂度，则并行效率低，可能不值得实施

5. **注意适用范围**：
   - 本方法基于理想PRAM模型（忽略通信、同步开销）
   - 实际GPU或分布式系统需额外考虑非理想因素
