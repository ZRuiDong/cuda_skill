# 3.4 CUDA 编程模型

## 一、本节作用

本节用于整理 CUDA 编程模型的核心知识。CUDA 编程模型是 CUDA 程序设计的基础，后续编写向量加法、矩阵计算、图像处理、归约、滤波和性能优化程序时，都需要先明确 CUDA 程序如何在 CPU 与 GPU 之间协同执行，以及线程如何映射到数据。

在 CUDA 程序中，CPU 通常称为主机端 host，GPU 通常称为设备端 device。一个完整的 CUDA 程序通常由两部分组成：运行在 CPU 上的 host code，以及运行在 GPU 上的 device code。host code 负责数据准备、显存分配、数据传输、kernel 启动和结果回收；device code 负责大规模并行计算。

CUDA 编程模型的核心思想是：将适合并行计算的任务写成 kernel function，由 host 端启动该 kernel，然后在 device 端由大量线程并行执行。

---

## 二、核心概念

### 1. host

host 指主机端，通常是 CPU 及其主存环境。

host 端主要负责：

1. 执行串行控制逻辑。
2. 分配和初始化输入数据。
3. 使用 `cudaMalloc` 在 GPU 上分配显存。
4. 使用 `cudaMemcpy` 在 host 与 device 之间传输数据。
5. 配置 grid 和 block。
6. 启动 kernel。
7. 从 device 端取回计算结果。
8. 释放资源。

在 CUDA 程序中，普通 C/C++ 代码默认运行在 host 端。

---

### 2. device

device 指设备端，通常是 NVIDIA GPU 及其显存环境。

device 端主要负责执行并行计算任务。GPU 适合处理具有大量相同或相似操作的数据并行任务，例如：

1. 向量加法。
2. 矩阵加法。
3. 矩阵乘法。
4. 图像滤波。
5. 并行归约。
6. 大规模数组计算。

device 端不适合处理复杂分支控制较多、并行粒度较低或线程间依赖严重的任务。

---

### 3. kernel function

kernel function，即内核函数，是在 GPU 上并行执行的函数。

CUDA 中使用 `__global__` 修饰的函数就是 kernel function。例如：

```cpp
__global__ void vectorAddKernel(const float* A, const float* B, float* C, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        C[idx] = A[idx] + B[idx];
    }
}
```

kernel function 具有以下特点：

1. 使用 `__global__` 修饰。
2. 在 device 端执行。
3. 由 host 端调用。
4. 返回值必须是 `void`。
5. 通过 `<<<grid, block>>>` 语法启动。
6. 一次 kernel 启动会创建大量 CUDA 线程。
7. 每个线程执行同一份 kernel 代码，但可以根据自身线程编号处理不同数据。

kernel 的典型调用方式如下：

```cpp
vectorAddKernel<<<grid, block>>>(d_A, d_B, d_C, n);
```

---

### 4. grid

grid 是一次 kernel 启动时创建的线程块集合。

一个 grid 由多个 block 组成。grid 可以是一维、二维或三维的，具体维度由 `dim3 grid` 设置。

例如，一维 grid：

```cpp
dim3 grid(128);
```

二维 grid：

```cpp
dim3 grid(64, 64);
```

在 kernel 内部，可以通过 `gridDim` 获取 grid 的维度信息。

---

### 5. block

block 是线程块，是 CUDA 线程组织的基本调度单位。

一个 block 由多个 thread 组成。同一个 block 内的线程可以：

1. 通过 shared memory 共享数据。
2. 使用 `__syncthreads()` 进行同步。
3. 协作完成局部计算任务。

不同 block 之间通常不能直接同步，也不能直接共享 shared memory。

block 可以是一维、二维或三维的，具体维度由 `dim3 block` 设置。

例如，一维 block：

```cpp
dim3 block(256);
```

二维 block：

```cpp
dim3 block(16, 16);
```

---

### 6. thread

thread 是 CUDA 中最基本的执行单元。

每个 thread 都会执行 kernel 函数中的代码。不同 thread 通过不同的索引值区分自己负责的数据位置。

例如，在一维数组处理中，通常让每个线程处理一个数组元素：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;

if (idx < n) {
    C[idx] = A[idx] + B[idx];
}
```

这里的 `idx` 就是当前线程对应的数据下标。

---

### 7. threadIdx

`threadIdx` 是 CUDA 内置变量，用于表示当前线程在线程块 block 内的索引。

它是一个三维变量，包含：

```cpp
threadIdx.x
threadIdx.y
threadIdx.z
```

对于一维 block，通常只使用 `threadIdx.x`。

对于二维图像或矩阵计算，通常使用：

```cpp
threadIdx.x
threadIdx.y
```

---

### 8. blockIdx

`blockIdx` 是 CUDA 内置变量，用于表示当前 block 在 grid 中的索引。

它也是一个三维变量，包含：

```cpp
blockIdx.x
blockIdx.y
blockIdx.z
```

一维数据通常使用 `blockIdx.x`。

二维数据通常使用 `blockIdx.x` 和 `blockIdx.y`。

---

### 9. blockDim

`blockDim` 是 CUDA 内置变量，用于表示每个 block 中线程的维度。

常用形式如下：

```cpp
blockDim.x
blockDim.y
blockDim.z
```

例如：

```cpp
dim3 block(256);
```

则在 kernel 中：

```cpp
blockDim.x == 256
```

---

### 10. gridDim

`gridDim` 是 CUDA 内置变量，用于表示 grid 中 block 的维度。

常用形式如下：

```cpp
gridDim.x
gridDim.y
gridDim.z
```

在实际编程中，`gridDim` 使用频率低于 `threadIdx`、`blockIdx` 和 `blockDim`，但在需要遍历整个 grid 或实现 grid-stride loop 时非常有用。

## 三、CUDA 程序基本流程

一个典型 CUDA 程序通常包含以下步骤：

1. 在 host 端分配并初始化输入数据。
2. 在 device 端分配显存。
3. 将输入数据从 host 端复制到 device 端。
4. 设置 `block` 和 `grid`。
5. 启动 kernel。
6. 等待 kernel 执行完成并检查错误。
7. 将结果从 device 端复制回 host 端。
8. 在 host 端验证计算结果。
9. 释放 device 端显存。
10. 释放 host 端资源。

典型流程代码如下：

```cpp
// 1. 分配 device memory
float *d_A = nullptr;
float *d_B = nullptr;
float *d_C = nullptr;

cudaMalloc((void**)&d_A, n * sizeof(float));
cudaMalloc((void**)&d_B, n * sizeof(float));
cudaMalloc((void**)&d_C, n * sizeof(float));

// 2. host to device
cudaMemcpy(d_A, h_A, n * sizeof(float), cudaMemcpyHostToDevice);
cudaMemcpy(d_B, h_B, n * sizeof(float), cudaMemcpyHostToDevice);

// 3. 设置 grid 和 block
dim3 block(256);
dim3 grid((n + block.x - 1) / block.x);

// 4. 启动 kernel
vectorAddKernel<<<grid, block>>>(d_A, d_B, d_C, n);

// 5. device to host
cudaMemcpy(h_C, d_C, n * sizeof(float), cudaMemcpyDeviceToHost);

// 6. 释放 device memory
cudaFree(d_A);
cudaFree(d_B);
cudaFree(d_C);
```

在正式程序中，应加入 CUDA 错误检查，而不是直接忽略 API 返回值。

## 四、CUDA C 语言扩展

CUDA C 是在 C/C++ 基础上加入少量扩展形成的编程方式。这些扩展主要用于区分函数运行位置、变量存储位置和 kernel 启动方式。

---

### 1. 函数类型限定符

#### `__global__`

`__global__` 用于声明 kernel function。

特点：

1. 在 device 端执行。
2. 由 host 端调用。
3. 返回值必须是 `void`。
4. 调用时必须使用 `<<<grid, block>>>` 语法。

示例：

```cpp
__global__ void kernelFunction() {
    // device code
}
```

---

#### `__device__`

`__device__` 用于声明只能在 device 端调用和执行的函数。

示例：

```cpp
__device__ float square(float x) {
    return x * x;
}
```

`__device__` 函数通常作为 kernel 内部的辅助函数。

---

#### `__host__`

`__host__` 用于声明在 host 端执行的函数。普通 C/C++ 函数默认就是 `__host__` 函数，因此通常不需要显式书写。

示例：

```cpp
__host__ void cpuFunction() {
    // host code
}
```

---

#### `__host__ __device__`

有些函数既希望在 CPU 上调用，也希望在 GPU 上调用，可以同时使用 `__host__` 和 `__device__` 修饰。

示例：

```cpp
__host__ __device__ float add(float a, float b) {
    return a + b;
}
```

这种写法常用于简单数学函数或小型工具函数。

---

### 2. 变量类型限定符

#### `__device__`

`__device__` 可以用于声明位于 device 全局存储器中的变量。

示例：

```cpp
__device__ int deviceCounter;
```

---

#### `__shared__`

`__shared__` 用于声明 shared memory 变量。

shared memory 是 block 内线程共享的高速存储器。

示例：

```cpp
__global__ void kernel() {
    __shared__ float tile[256];

    int tid = threadIdx.x;
    tile[tid] = 0.0f;

    __syncthreads();
}
```

注意：shared memory 只在同一个 block 内共享，不同 block 之间不能共享同一块 shared memory。

---

#### `__constant__`

`__constant__` 用于声明 constant memory 变量。

constant memory 适合存储小规模、只读、被大量线程重复访问的数据。

示例：

```cpp
__constant__ float filter[9];
```

在图像滤波中，卷积核、滤波模板等小型只读数组适合放入 constant memory。

---

## 五、kernel 启动语法

CUDA kernel 使用如下语法启动：

```cpp
kernelName<<<grid, block>>>(arguments);
```

完整形式为：

```cpp
kernelName<<<grid, block, sharedMemSize, stream>>>(arguments);
```

其中：

1. `grid` 表示 grid 的维度，即启动多少个 block。
2. `block` 表示每个 block 中有多少个 thread。
3. `sharedMemSize` 表示动态 shared memory 大小，单位是字节，可省略。
4. `stream` 表示 CUDA stream，可省略。

常用写法如下：

```cpp
int threadsPerBlock = 256;
int blocksPerGrid = (n + threadsPerBlock - 1) / threadsPerBlock;

vectorAddKernel<<<blocksPerGrid, threadsPerBlock>>>(d_A, d_B, d_C, n);
```

也可以使用 `dim3`：

```cpp
dim3 block(256);
dim3 grid((n + block.x - 1) / block.x);

vectorAddKernel<<<grid, block>>>(d_A, d_B, d_C, n);
```

## 六、线程映射方法

线程映射是 CUDA 编程中最重要的设计步骤之一。它决定每个线程负责处理哪一部分数据。

---

### 1. 一维线程映射

一维数组最常用的线程映射方式如下：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;
```

含义是：

1. `blockIdx.x` 表示当前 block 的编号。
2. `blockDim.x` 表示每个 block 中线程数量。
3. `threadIdx.x` 表示当前线程在 block 内的编号。
4. 三者组合得到当前线程在整个 grid 中的全局编号。

示例：

```cpp
__global__ void vectorAddKernel(const float* A, const float* B, float* C, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        C[idx] = A[idx] + B[idx];
    }
}
```

边界判断 `if (idx < n)` 很重要，因为启动的线程总数通常会略大于数据规模。

---

### 2. 二维线程映射

二维数据常见于矩阵和图像。二维线程映射方式如下：

```cpp
int x = blockIdx.x * blockDim.x + threadIdx.x;
int y = blockIdx.y * blockDim.y + threadIdx.y;
```

对于行优先存储的二维数组，线性下标通常为：

```cpp
int idx = y * width + x;
```

示例：

```cpp
__global__ void matrixAddKernel(const float* A, const float* B, float* C,
                                int width, int height) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;

    if (x < width && y < height) {
        int idx = y * width + x;
        C[idx] = A[idx] + B[idx];
    }
}
```

对应的 kernel 启动配置：

```cpp
dim3 block(16, 16);
dim3 grid((width + block.x - 1) / block.x,
          (height + block.y - 1) / block.y);

matrixAddKernel<<<grid, block>>>(d_A, d_B, d_C, width, height);
```

---

### 3. 三维线程映射

三维线程映射常见于体数据处理、三维网格计算、体绘制等任务。

```cpp
int x = blockIdx.x * blockDim.x + threadIdx.x;
int y = blockIdx.y * blockDim.y + threadIdx.y;
int z = blockIdx.z * blockDim.z + threadIdx.z;
```

三维数组线性下标常写为：

```cpp
int idx = (z * height + y) * width + x;
```

示例：

```cpp
if (x < width && y < height && z < depth) {
    int idx = (z * height + y) * width + x;
    data[idx] = 0.0f;
}
```

---

### 4. grid-stride loop

当数据规模很大，或者希望一个 kernel 适配任意数据规模时，可以使用 grid-stride loop。

```cpp
for (int idx = blockIdx.x * blockDim.x + threadIdx.x;
     idx < n;
     idx += blockDim.x * gridDim.x) {
    C[idx] = A[idx] + B[idx];
}
```

这种写法的优点是：

1. kernel 对数据规模更灵活。
2. 线程可以处理多个元素。
3. 可以减少对 grid 大小的严格依赖。
4. 常用于大型数组处理。

---

## 七、边界判断

CUDA kernel 中通常必须进行边界判断。

原因是 grid 和 block 的大小一般按照整除向上取整计算。例如：

```cpp
dim3 block(256);
dim3 grid((n + block.x - 1) / block.x);
```

如果 `n` 不是 256 的整数倍，那么最后一个 block 中会有部分线程对应的 `idx >= n`。这些线程如果不加判断，就会访问数组越界。

正确写法：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;

if (idx < n) {
    C[idx] = A[idx] + B[idx];
}
```

二维数据也需要判断：

```cpp
if (x < width && y < height) {
    int idx = y * width + x;
    C[idx] = A[idx] + B[idx];
}
```

## 八、执行结构

CUDA 程序的执行结构可以理解为：

```text
host code
    |
    | 启动 kernel
    v
grid
    |
    +-- block 0
    |      +-- thread 0
    |      +-- thread 1
    |      +-- thread 2
    |
    +-- block 1
    |      +-- thread 0
    |      +-- thread 1
    |      +-- thread 2
    |
    +-- block 2
           +-- thread 0
           +-- thread 1
           +-- thread 2
```

一个 kernel 对应一个 grid。

一个 grid 包含多个 block。

一个 block 包含多个 thread。

每个 thread 都执行同一份 kernel 代码，但通过 `threadIdx`、`blockIdx` 等内置变量处理不同数据。

---

## 九、内核函数设计原则

设计 kernel function 时，应考虑以下问题：

1. 每个线程处理什么数据。
2. 是否需要边界判断。
3. 数据是否连续访问。
4. 是否需要 shared memory。
5. 是否需要线程同步。
6. 是否存在分支发散。
7. 是否存在写冲突。
8. 是否需要原子操作。
9. 是否存在数组越界风险。
10. 是否能减少 host-device 数据传输。

一个好的 kernel 通常应满足：

1. 线程数量足够多。
2. 每个线程任务相对简单。
3. 线程之间依赖较少。
4. global memory 访问尽量连续。
5. 尽量避免不必要的同步。
6. 尽量避免同一 warp 内严重分支发散。

## 十、存储器结构与线程映射关系

CUDA 中不同存储器的作用范围不同。

| 存储器类型 | 访问范围 | 典型用途 |
|---|---|---|
| register | 单个线程私有 | 保存线程内部临时变量 |
| local memory | 单个线程私有 | 寄存器不足或局部数组溢出时使用 |
| shared memory | 同一个 block 内线程共享 | block 内数据复用、局部归约、分块计算 |
| global memory | 所有线程可访问 | 大规模输入输出数组 |
| constant memory | 所有线程只读访问 | 小规模只读参数、卷积核、查找表 |
| texture memory | 只读访问，带缓存特性 | 图像、空间局部性较强的数据 |

在编写 kernel 时，需要根据线程映射方式选择合适的存储器。

例如：

1. 向量加法通常只需要 global memory。
2. 矩阵乘法基础版本使用 global memory，优化版本使用 shared memory。
3. 图像卷积的滤波模板可以使用 constant memory。
4. 图像或体数据采样可以考虑 texture memory。
5. block 内归约通常使用 shared memory。

## 十一、线程通信机制

CUDA 中常见通信方式主要有以下几种。

---

### 1. 通过 global memory 通信

所有线程都可以访问 global memory，因此 global memory 可以用于不同 block 之间的数据传递。

但需要注意：

1. global memory 访问延迟较高。
2. 不同 block 之间不能在同一个 kernel 内进行全局同步。
3. 如果多个线程写同一地址，可能产生 race condition。
4. 必要时需要使用 atomic operation。

---

### 2. 通过 shared memory 通信

同一个 block 内的线程可以通过 shared memory 共享数据。

典型写法：

```cpp
__shared__ float tile[256];

int tid = threadIdx.x;
tile[tid] = input[blockIdx.x * blockDim.x + tid];

__syncthreads();

float value = tile[tid];
```

`__syncthreads()` 用于保证同一个 block 中所有线程都执行到该位置后再继续向下执行。

注意：`__syncthreads()` 只能同步同一个 block 内的线程，不能同步不同 block。

---

### 3. 通过 atomic operation 避免写冲突

当多个线程可能同时更新同一个地址时，需要使用原子操作。

例如：

```cpp
atomicAdd(result, value);
```

原子操作可以保证更新的正确性，但通常会带来额外开销。因此，在优化时应尽量减少不必要的原子操作。

---

## 十二、同步机制

### 1. block 内同步

CUDA 中最常用的同步函数是：

```cpp
__syncthreads();
```

它的作用是同步同一个 block 内的所有线程。

常见使用场景：

1. shared memory 写入后，确保所有线程都写完再读取。
2. block 内归约每轮计算之后同步。
3. 分块矩阵乘法中，每轮 tile 加载完成后同步。

错误示例：

```cpp
if (threadIdx.x == 0) {
    __syncthreads();
}
```

这种写法是错误的，因为不是所有线程都会执行到 `__syncthreads()`，可能导致死锁。

正确原则是：同一个 block 中所有线程必须以一致的控制流执行到同一个 `__syncthreads()`。

---

### 2. host 与 device 同步

kernel 启动通常是异步的。也就是说，host 端启动 kernel 后，不一定会等待 kernel 完成才继续执行后续语句。

如果需要等待 GPU 执行完成，可以使用：

```cpp
cudaDeviceSynchronize();
```

常见场景：

1. 检查 kernel 执行错误。
2. 测量 kernel 执行时间。
3. 在 host 端读取结果前确保 kernel 完成。
4. 调试 CUDA 程序。

## 十三、CUDA 软件体系

在 CUDA 编程中，常见软件组成包括：

1. CUDA C/C++ 编程语言扩展。
2. CUDA Runtime API。
3. CUDA Driver API。
4. `nvcc` 编译器。
5. CUDA 标准库和数学函数。
6. 调试与性能分析工具。
7. GPU 驱动程序。

初学者通常优先使用 CUDA Runtime API，因为它更接近普通 C/C++ 编程习惯，使用方式也更简单。

常见 Runtime API 包括：

```cpp
cudaMalloc
cudaMemcpy
cudaFree
cudaMemset
cudaDeviceSynchronize
cudaGetLastError
cudaGetDeviceCount
cudaGetDeviceProperties
```

## 十四、典型完整代码：向量加法

下面是一个最小但完整的 CUDA 向量加法示例，可用于说明 CUDA 编程模型。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cmath>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void vectorAddKernel(const float* A, const float* B, float* C, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        C[idx] = A[idx] + B[idx];
    }
}

int main() {
    const int n = 1 << 20;
    const size_t bytes = n * sizeof(float);

    float* h_A = static_cast<float*>(malloc(bytes));
    float* h_B = static_cast<float*>(malloc(bytes));
    float* h_C = static_cast<float*>(malloc(bytes));

    if (h_A == nullptr || h_B == nullptr || h_C == nullptr) {
        fprintf(stderr, "Host memory allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int i = 0; i < n; ++i) {
        h_A[i] = static_cast<float>(i);
        h_B[i] = static_cast<float>(2 * i);
    }

    float* d_A = nullptr;
    float* d_B = nullptr;
    float* d_C = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_A, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_B, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_C, bytes));

    CUDA_CHECK(cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice));
    CUDA_CHECK(cudaMemcpy(d_B, h_B, bytes, cudaMemcpyHostToDevice));

    dim3 block(256);
    dim3 grid((n + block.x - 1) / block.x);

    vectorAddKernel<<<grid, block>>>(d_A, d_B, d_C, n);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < n; ++i) {
        float expected = h_A[i] + h_B[i];
        if (fabs(h_C[i] - expected) > 1e-5f) {
            correct = false;
            fprintf(stderr, "Mismatch at %d: got %f, expected %f\n",
                    i, h_C[i], expected);
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaFree(d_A));
    CUDA_CHECK(cudaFree(d_B));
    CUDA_CHECK(cudaFree(d_C));

    free(h_A);
    free(h_B);
    free(h_C);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

编译命令：

```bash
nvcc -O2 main.cu -o main
```

运行命令：

```bash
./main
```

预期输出：

```text
Result: PASS
```

## 十五、常见错误

### 1. 忘记边界判断

错误写法：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;
C[idx] = A[idx] + B[idx];
```

当线程总数大于数组长度时，该写法会导致越界访问。

正确写法：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;

if (idx < n) {
    C[idx] = A[idx] + B[idx];
}
```

---

### 2. `cudaMalloc` 分配大小错误

错误写法：

```cpp
cudaMalloc((void**)&d_A, n);
```

这里的 `n` 是元素个数，不是字节数。

正确写法：

```cpp
cudaMalloc((void**)&d_A, n * sizeof(float));
```

---

### 3. `cudaMemcpy` 方向错误

host 到 device：

```cpp
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
```

device 到 host：

```cpp
cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost);
```

方向写反会导致结果错误，甚至程序崩溃。

---

### 4. host pointer 和 device pointer 混用

错误示例：

```cpp
vectorAddKernel<<<grid, block>>>(h_A, h_B, h_C, n);
```

kernel 在 device 端执行，不能直接使用普通 host pointer。

正确写法：

```cpp
vectorAddKernel<<<grid, block>>>(d_A, d_B, d_C, n);
```

---

### 5. kernel 启动后不检查错误

错误写法：

```cpp
kernel<<<grid, block>>>(...);
```

更推荐写法：

```cpp
kernel<<<grid, block>>>(...);
CUDA_CHECK(cudaGetLastError());
CUDA_CHECK(cudaDeviceSynchronize());
```

这样可以及时发现 kernel 启动参数错误、非法内存访问等问题。

---

### 6. 错误理解 block 与 thread 的关系

错误理解：

```text
一个 thread 包含多个 block。
```

正确理解：

```text
一个 grid 包含多个 block，一个 block 包含多个 thread。
```

层次关系如下：

```text
grid > block > thread
```

---

### 7. 在不同 block 之间使用 `__syncthreads()`

`__syncthreads()` 只能同步同一个 block 内的线程，不能同步不同 block。

如果需要不同 block 之间同步，通常需要：

1. 拆分为多个 kernel。
2. 在 host 端两个 kernel 调用之间形成全局同步。
3. 使用更高级的 CUDA 特性，但初学阶段通常不建议直接使用。

## 十六、回答模板

当用户询问 CUDA 编程模型时，可以按照以下方式回答：

```text
CUDA 程序由 host 端代码和 device 端代码组成。host 端通常由 CPU 执行，负责数据准备、显存分配、数据传输和 kernel 启动；device 端通常由 GPU 执行，负责大规模并行计算。CUDA 中在 GPU 上执行的函数称为 kernel function，使用 __global__ 修饰，并通过 <<<grid, block>>> 语法由 host 端启动。

一次 kernel 启动对应一个 grid，一个 grid 由多个 block 组成，一个 block 又由多个 thread 组成。每个线程执行同一份 kernel 代码，但可以通过 threadIdx、blockIdx、blockDim 和 gridDim 计算自己的全局编号，从而处理不同的数据元素。
```

当用户要求写 CUDA 程序时，可以按照以下方式组织：

```text
首先分析串行算法中每个数据元素的计算是否相互独立。如果每个元素可以独立计算，就可以让一个 CUDA thread 负责一个元素。然后根据数据规模设置 block 和 grid，使用 cudaMalloc 分配 device memory，使用 cudaMemcpy 将输入数据复制到 GPU，启动 kernel 完成并行计算，最后将结果复制回 host 并验证正确性。
```

## 十八、小结

CUDA 编程模型的核心是 CPU 与 GPU 的协同工作。CPU 作为 host 负责串行控制和任务调度，GPU 作为 device 负责大规模并行计算。CUDA 通过 kernel function 将并行任务交给 GPU 执行，而 kernel 的执行线程按照 grid、block 和 thread 三个层次组织。

在实际编写 CUDA 程序时，最重要的是明确每个线程负责处理哪个数据元素，并正确设置 grid 和 block。同时还需要注意 host 与 device 之间的数据传输、device memory 的分配与释放、kernel 启动后的错误检查，以及数组边界判断。

掌握本节内容后，就可以开始编写最基础的 CUDA 程序，例如 HelloWorld、向量加法、矩阵加法和简单图像处理程序。
