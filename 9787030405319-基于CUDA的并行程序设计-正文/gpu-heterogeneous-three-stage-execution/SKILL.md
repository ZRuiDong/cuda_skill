# GPU异构系统三阶段执行流程

## 何时使用
- 输入数据存储在CPU主存中，需在GPU上执行计算
- 系统采用分离式内存架构（CPU主存与GPU显存物理隔离）
- 使用CUDA、OpenCL等要求显式数据传输的编程模型
- **不适用**于统一内存架构（如某些现代APU）中自动数据迁移的场景

## 执行步骤

### 1. 数据传输阶段（Host → Device）
- 将输入数据从CPU主存显式复制到GPU显存
- 使用CUDA运行时API（如`cudaMemcpy()`）或OpenCL等效函数
- **优化建议**：尽量减少传输数据量和频次，避免PCI-E带宽成为瓶颈

### 2. GPU计算阶段（Kernel Execution）
- CPU启动GPU核函数（kernel），GPU以SIMT方式并行执行
- 利用GPU内部高带宽存储层次（寄存器、共享内存）优化性能
- 此阶段CPU可异步执行其他任务（若使用异步流）

### 3. 结果回传阶段（Device → Host）
- 将计算结果从GPU显存复制回CPU主存
- 同样通过显式API调用完成
- **注意**：若后续操作仍在GPU上进行，应跳过此步骤以避免冗余数据移动

## 关键约束
- 整个流程依赖显式内存拷贝，不能自动共享数据
- 若数据已在显存中，可跳过第一阶段
- 显存容量限制单次可处理的数据规模
- PCI-E带宽直接影响数据传输阶段耗时
