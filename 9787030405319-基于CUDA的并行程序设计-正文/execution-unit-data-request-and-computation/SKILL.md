# 执行单元数据请求与计算流程

## 何时使用
- 执行单元已由本地任务管理器分派任务
- 任务描述包含操作逻辑但不含实际数据（例如“计算A、B矩阵第一行前10个元素相加”）
- 数据已在内存或外存中注册，但执行单元尚未获取其物理地址

## 执行步骤
1. **接收任务描述**：解析任务代码，识别操作类型与数据范围（如“整行计算”或“前10个元素”）。
2. **发送数据地址请求**：向本节点的数据服务模块发送符合表9.1规范的请求，明确所需数据的逻辑标识。
3. **阻塞等待地址返回**：在收到数据服务模块返回的物理地址（含节点ID、存储类型、偏移量等）前，执行单元必须保持阻塞状态。
4. **选择执行资源**：
   - 若为CPU执行单元，使用CPU线程处理数据；
   - 若为GPU执行单元，启动GPU线程，形成CPU-GPU混合执行模式。
5. **完成计算并通知**：计算结束后，立即向本地任务管理器发送“任务完成”消息。
6. **准备接收新任务**：本地任务管理器收到通知后可分派后续任务；执行单元不缓存数据地址，每次任务均重新请求以确保数据一致性。

> **注意**：执行单元不得直接访问原始数据，且每次任务必须重新请求地址，禁止复用历史地址。
