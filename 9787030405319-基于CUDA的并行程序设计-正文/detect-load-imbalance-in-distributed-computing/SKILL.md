# 检测分布式计算中的负载不均衡

## 何时使用
- 系统已完成多轮迭代计算
- 各节点内存使用状态可被监控
- 任务已被划分为子矩阵
- 观察到某节点在多轮迭代中 consistently 先于其他节点完成计算任务
- 怀疑性能瓶颈源于任务分配不均（而非网络延迟或硬件故障）

## 执行步骤
1. **监控内存装载曲线**：获取各计算节点在迭代过程中的内存使用时间序列数据（memory_curve）。
2. **识别提前完成节点**：观察是否存在某节点的内存曲线在每轮SpMM计算结束后率先变为水平（表示停止从外存读取数据，进入Dot计算阶段）。
3. **验证重复性**：确认该现象在连续多轮迭代中重复出现（例如node1在第一轮和第二轮均率先完成）。
4. **核查任务划分**：检查任务划分表，对比各节点分配的子矩阵数量（submatrix_count_per_node）。若某节点分配数量明显少于其他节点（如node1分配10个，其余分配11个），则确认为任务量不均。
5. **生成告警**：将该时间差标注为“负载不均衡点(load imbalance)”，并记录其对整体同步效率的影响，用于后续优化任务调度策略。

## 关键变量
- `N_max`：控制矩阵划分粒度的整数参数
- `submatrix_count_per_node`：每个节点分配到的子矩阵数量数组
- `memory_curve`：各节点内存使用随时间变化的时间序列数据

> 注意：本技能仅适用于任务划分不均引起的性能差异，不适用于由网络延迟或硬件故障导致的非对称行为。
