# 并行加速比评估方法

## 何时使用
当满足以下条件时应用本技能：
- 已知串行执行时间（`t_seq`）或单处理器在并行系统中的执行时间（`t_1`）
- 已知P个处理器的并行执行时间（`t_P`）
- 问题规模固定
- 需要量化并行化带来的性能收益

## 执行步骤
1. **确认基准类型**：
   - 若比较对象为串行操作系统下的执行时间，使用串行机加速比
   - 若比较对象为并行操作系统下单处理器的执行时间，使用单处理器加速比

2. **计算加速比**：
   - 单处理器加速比：`S_P = t_1 / t_P`
   - 串行机加速比：`Ŝ_P = t_seq / t_P`

3. **解释结果**：
   - 加速比值越大，表示并行效率越高
   - 通常 `Ŝ_P < S_P`，因为并行操作系统存在额外开销

> 注意：本技能不适用于无明确串行基准的问题场景。
