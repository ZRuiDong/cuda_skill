# MIMD多处理机系统分类与架构选型

## 触发条件
当构建多处理器并行系统，且需根据内存组织方式选择对称多处理（SMP）、分布式共享存储器（DSM）或大规模并行处理机（MPP）架构时，启用本技能。

## 执行步骤

### 步骤1：判定内存组织方式
检查系统的内存组织结构：
- **共享物理地址空间**：所有处理器访问同一物理内存 → 考虑 **SMP**
- **逻辑统一但物理分布的地址空间**：各处理器有本地内存，但通过软件/硬件实现统一虚拟地址 → 考虑 **DSM**
- **完全物理分布式内存**：节点间无共享地址空间，依赖显式消息传递 → 考虑 **MPP**

### 步骤2：评估系统规模与性能需求
- 若处理器数量 **< 64** 且要求低延迟共享内存访问 → **推荐 SMP**
- 若为中等规模（64–数百核），且应用具有强数据局部性 → **推荐 DSM**
- 若需超大规模（数百至数千核）科学计算，追求 TeraFLOPs 级算力、TB 级内存与 I/O → **推荐 MPP**

### 步骤3：验证约束条件
- 排除 SIMD 专用加速器（如传统 GPU）
- 若采用 SMP，确认总线或互连带宽可支撑目标处理器数量
- 若采用 DSM 或 MPP，确保网络互连具备足够带宽与低延迟

### 步骤4：输出推荐结果
返回以下内容：
- 推荐的 MIMD 子类型（SMP / DSM / MPP）
- 关键配置参数（如互连类型、一致性协议、通信模型）
- 适用场景说明

## 输入变量
- `memory_organization`：内存组织方式（`shared_physical` / `distributed_logical` / `distributed_physical`）
- `processor_count`：处理器数量（整数）
- `communication_model`：通信模型（`shared_memory` / `message_passing`）

## 输出
返回推荐的 MIMD 子类型及其关键配置参数。
