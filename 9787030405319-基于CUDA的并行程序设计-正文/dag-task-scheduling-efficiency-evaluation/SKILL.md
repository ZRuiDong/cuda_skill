# DAG任务调度效率评估

## 何时使用
- 系统已运行完任务创建与分配阶段
- 各阶段耗时（任务创建、任务分配、总运行时间）可被精确测量
- 使用的是D0oC+LAF中间件与DAG任务调度架构
- 不适用于传统MPI手动任务划分场景

## 执行步骤
1. **测量任务创建时间**：记录LAF对用户代码进行翻译并生成任务描述所消耗的时间（秒）。
2. **测量任务分配时间**：记录全局任务管理器构建DAG并将任务分发至各计算节点所消耗的时间（秒）。
3. **获取总运行时间**：记录从任务提交到执行完成的完整系统运行时间（秒）。
4. **计算调度开销占比**：
   - 调度开销 = (任务创建时间 + 任务分配时间) / 总运行时间
5. **应用判定规则**：
   - 若任务创建时间 < 0.1 秒 **且** 任务分配时间 ≤ 总时间的10%，则判定为高效调度。
   - 更一般地，只要调度开销远小于I/O时间和计算时间（例如调度开销占比显著低于50%），即可认为调度开销可忽略，并行效率高。
6. **输出结果**：返回调度开销占比数值及效率判定结论。

> 注意：即使在大规模节点（如29节点）下任务分配时间较长（如61秒），只要仍显著小于I/O（如263秒）和计算时间（如277秒），仍视为高效。
