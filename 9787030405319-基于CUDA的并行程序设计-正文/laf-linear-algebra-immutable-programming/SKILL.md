# LAF线性代数不可变编程约束

## 何时使用
- 使用LAF框架进行分布式线性代数计算时
- 调用表9.3所列原语（如addM、axpyV、dot等）操作矩阵、向量或标量时
- 需要确保任务图构建正确性和并行调度安全性的场景

## 如何执行
1. **将所有数据对象视为不可变**：包括稠密矩阵、稀疏矩阵、向量和标量，一旦创建不得修改。
2. **每次原语调用必须生成全新结果对象**：
   - 示例：`addM(A, B)` 必须返回一个全新的矩阵 `C = A + B`
   - 示例：`axpyV(a, x, b)` 必须返回一个全新的向量 `y = a*x + b`
3. **禁止任何形式的后续修改**：不得对已存在的对象进行元素赋值、结构调整或内容覆盖。
4. **利用DOoC中间件兼容的数据块分布机制**：数据自动分块存储于集群节点，无需手动管理位置。
5. **验证代码是否违反约束**：若尝试修改已有对象，应立即抛出编程错误；正确做法是始终创建新对象。

> ⚠️ 违反此约束将导致任务调度失败或数据不一致，务必在编写LAF程序时严格遵守。
