# DataCutter 分布式流处理架构机制

## 何时使用
- 需要构建基于 MPI 通信协议的分布式流式数据处理系统
- 目标环境为支持多线程与 GPU 的 CPU/GPU 混合计算集群
- 任务依赖 DataCutter 中间件运行时环境

## 执行步骤

1. **定义基本单元**：将每个数据流处理逻辑封装为一个“过滤器”（filter），每个 filter 占用一个独立线程。

2. **准备数据表示**：所有输入/输出数据以 untyped 格式存储于“数据缓存块”（databuffer）中，便于统一管理与跨设备传输。

3. **配置通信接口**：为每个 filter 添加一个或多个“抽象数据流接口”（port），用于接收上游数据或向下游发送结果。

4. **构建实例逻辑**：通过连接多个 filter 的 port，形成完整的 DataCutter 实例，明确数据流向与处理顺序。

5. **启用并行机制**：
   - 复制 filter 以实现任务级并行；
   - 在同一节点上同时部署 CPU 线程版和 GPU 调用版的同一 filter（例如 node 1 中的 B 和 C）；
   - 上游 filter 输出的数据自动分发至所有连接的下游 filter 并行处理。

6. **生成部署视图**：
   - 绘制**布局图**（layout）：展示 filter、port 连接关系及逻辑数据流（如 A→B→C/D→E）；
   - 生成**实现图**（placement）：标明每个 filter 在物理节点、CPU 核心或 GPU 上的实际分配位置。

7. **启动数据流动**：按布局图顺序驱动数据流经各 filter，完成端到端流式计算。

> 注意：本技能仅适用于流式数据处理任务，且必须在已安装 DataCutter 中间件的 MPI 环境中执行。
