# CPU-GPU任务分配原则

## 何时使用
- 存在由多核CPU和众核GPU组成的异构硬件平台；
- 待优化的应用程序可被划分为串行逻辑控制部分与数据并行计算部分；
- 计算任务包含高计算密度（浮点运算量远大于内存访问量）、低控制流复杂度（无频繁条件跳转、递归等）的子任务。

> **注意**：不适用于强分支依赖、复杂控制流或无法分解为独立子任务的问题。

## 执行步骤

1. **分析应用程序的计算特征**：识别程序中的串行逻辑控制部分与数据并行计算部分。

2. **将满足以下条件的子任务分配给GPU执行**：
   - 子任务之间相互独立或依赖性很小；
   - 不需要复杂的控制流（如频繁的条件跳转、递归等）；
   - 具有高计算密度（单位数据访问对应的浮点运算次数高）。

3. **将以下任务保留在CPU上执行**：
   - 涉及复杂逻辑判断、事务处理、I/O操作或操作系统交互的部分；
   - 控制整个程序流程的主循环和调度逻辑。

4. **最小化CPU与GPU间的数据传输开销**：避免频繁的主机-设备内存拷贝，尽可能复用设备端数据。

5. **遵循核心原则**：GPU适合处理“耗时较长且能并行执行的循环结构”，而CPU擅长处理具有空间/时间局部性或不规则结构的工作负载。

## 输出结果
生成一份明确的CPU-GPU任务划分方案，指明哪些函数、循环或代码块应由GPU执行，哪些应保留在CPU上。
