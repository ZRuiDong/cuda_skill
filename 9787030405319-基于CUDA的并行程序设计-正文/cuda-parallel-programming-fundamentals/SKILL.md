# CUDA并行程序设计基础

## 何时使用
当满足以下条件时应用本技能：
- 需要利用GPU加速计算密集型任务
- 开发环境已安装CUDA Toolkit
- 拥有支持CUDA的NVIDIA GPU
- 具备C/C++编程基础

## 核心执行流程
1. **理解CUDA程序结构**：
   - 主机（Host，CPU）负责数据准备、内存分配和内核启动
   - 设备（Device，GPU）执行并行计算的内核函数（Kernel）

2. **掌握线程层次结构**：
   - 线程（Thread）：最小执行单元
   - 线程块（Block）：协同工作的线程组，可共享内存
   - 网格（Grid）：一个或多个线程块的集合，对应一次核函数调用

3. **遵循基本编程流程**：
   a) 在主机端分配设备内存（`cudaMalloc`）
   b) 将数据从主机复制到设备（`cudaMemcpy`）
   c) 启动核函数（使用 `<<<gridDim, blockDim>>>` 语法）
   d) 将结果从设备复制回主机
   e) 释放设备内存（`cudaFree`）

4. **应用性能优化原则**：
   - 最大化并行度
   - 减少内存访问延迟
   - 避免分支发散（warp divergence）

> 注意：本技能不适用于纯CPU串行算法或非NVIDIA硬件平台。
