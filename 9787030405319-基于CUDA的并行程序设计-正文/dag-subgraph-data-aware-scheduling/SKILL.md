# DAG子图数据感知调度

## 何时使用
- 系统已完成启动阶段的外存扫描预处理
- 各节点数据服务模块已上报本地存储概要
- 全局任务管理器需要首次分派DAG子图任务
- **不适用于**动态新增数据或运行时任务重调度

## 执行步骤
1. **确认前提条件**：确保外存扫描已完成，且所有节点的数据服务模块已上报存储概要。
2. **构建数据位置索引**：
   - 使用数据块ID作为键，记录每个数据块所在的节点列表
   - 该索引由全局任务管理器维护
3. **应用调度策略分派DAG子图任务**：
   - **优先选择**持有最多输入数据的节点（遵循数据局部性原则）
   - **若多个节点输入数据量相近**，则选择当前负载指标（如任务队列长度或资源占用率）最低的节点（负载均衡原则）
   - **仅当无法满足前两条时**，才允许跨节点数据传输
4. **交付执行**：将任务分配方案下发至对应节点的本地任务管理器，由其负责本节点内执行单元的细粒度调度。
5. **输出结果**：返回优化后的任务-节点分配方案，目标是最小化数据移动并均衡计算负载。

> 注意：此策略仅用于首次任务分派，不可用于运行时动态调整。
