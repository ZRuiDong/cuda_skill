# CPU+GPU异构系统优势评估

在决定是否采用CPU+GPU异构架构时，请按以下三方面综合评估其优势：

## 1. 评估计算能力的互补性
- **GPU优势**：集成大量计算单元（如Radeon HD 5870含1600核心），适合规则、高密度的并行计算任务。
- **CPU优势**：基于高速缓存，擅长处理具有空间/时间局部性或不规则结构的任务（如指针遍历、稀疏图算法）。
- **组合价值**：两者结合可覆盖更广泛的应用谱系，形成“优良解决方案”。

## 2. 评估存储带宽对“存储墙”问题的缓解能力
- **CPU限制**：多核CPU内存带宽通常低于100GB/s，成为性能瓶颈。
- **GPU优势**：提供极高存储带宽（如Radeon HD 5870达154GB/s，Tesla系列最高177GB/s）。
- **显式管理**：支持程序员控制数据流动，使大规模数据并行计算高效利用带宽。

## 3. 评估性价比与能效比
- **关注指标**：单位资金、空间、功耗下的FLOPS（即“高生产率”）。
- **GPU优势**：因结构简单、机制高效、功耗低，通常显著优于纯CPU系统。
- **实证参考**：天河一号（6144 CPU + 5120 GPU）峰值1.206PFLOPS，Linpack实测563.1TFLOPS，能效比达4.3亿次/瓦。

## 决策建议
- **推荐采用**：若应用场景匹配GPU特性（大规模并行、规则计算）。
- **谨慎评估**：若应用为纯串行、强分支依赖、或对延迟敏感而非吞吐量，需重点评估数据传输开销是否抵消性能收益。
