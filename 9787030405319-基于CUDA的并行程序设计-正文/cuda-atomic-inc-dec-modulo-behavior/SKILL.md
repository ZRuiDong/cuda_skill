# CUDA原子函数atomicInc/atomicDec的模运算行为

## 何时使用
- 在CUDA内核中调用`atomicInc()`或`atomicDec()`函数时
- 需要对共享内存或全局内存中的`unsigned int`变量执行线程安全的循环递增或递减
- 实现环形缓冲区、循环计数器或带边界重置的同步机制

## 如何执行

### atomicInc 执行流程
1. 原子读取地址 `address` 处的32位无符号整数值 `old`
2. 计算新值：
   - 若 `old >= val`，则新值 = `0`
   - 否则，新值 = `old + 1`
3. 原子写入新值到同一地址
4. 返回原始值 `old`

### atomicDec 执行流程
1. 原子读取地址 `address` 处的32位无符号整数值 `old`
2. 计算新值：
   - 若 `(old == 0) OR (old > val)`，则新值 = `val`
   - 否则，新值 = `old - 1`
3. 原子写入新值到同一地址
4. 返回原始值 `old`

### 关键约束
- 仅支持 `unsigned int*` 类型的内存地址
- 不支持64位整型或有符号整数
- 仅适用于全局存储器和共享存储器
- 整个读-改-写过程为原子操作，确保多线程安全

> 提示：`atomicInc` 和 `atomicDec` 实现的是模 `(val + 1)` 的循环行为，常用于构建线程安全的环形结构。
