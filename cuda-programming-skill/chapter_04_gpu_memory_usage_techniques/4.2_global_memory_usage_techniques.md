# 4.2 全局存储器的使用技巧

## 一、本节作用

本节用于整理 CUDA 全局存储器 global memory 的使用方法和优化技巧。

在 CUDA 程序中，全局存储器是最常用的大容量 device memory。绝大多数输入数组、输出数组、图像数据、矩阵数据和中间结果都会存储在 global memory 中。host 端通常通过 `cudaMalloc`、`cudaMemcpy` 和 `cudaFree` 管理全局存储器，device 端 kernel 则通过指针直接访问全局存储器中的数据。

全局存储器容量大、使用灵活，但访问延迟较高。因此，CUDA 程序性能很大程度上取决于 global memory 的访问方式是否合理。全局存储器优化的核心目标是：

```text
减少不必要的 global memory 访问，并尽量让连续线程访问连续地址。
```

掌握本节内容后，应能够完成以下任务：

1. 正确使用 `cudaMalloc` 分配 global memory。
2. 正确使用 `cudaMemcpy` 在 host 与 device 之间传输数据。
3. 在 kernel 中正确访问 global memory。
4. 理解 coalesced access 合并访问。
5. 避免非连续访问、间隔访问和非对齐访问。
6. 判断结构体数组是否会造成非合并访问。
7. 使用 SoA 数据布局改善全局存储器访问。
8. 使用 `cudaMallocPitch` 处理二维数组对齐问题。
9. 分析 global memory 访问瓶颈。
10. 为后续 shared memory 优化和存储器访问优化打基础。

---

## 二、global memory 的基本特点

global memory，即全局存储器，位于 GPU 板载显存中。它是 CUDA 程序中最常使用的大容量存储器。

global memory 具有以下特点：

1. 容量较大。
2. 位于 GPU 片外显存。
3. 所有 CUDA 线程都可以访问。
4. host 端可以通过 CUDA Runtime API 读写。
5. 可跨 kernel 保持数据。
6. 访问延迟较高。
7. 带宽较高，但需要合理访问才能充分利用。
8. 适合存储大规模输入和输出数据。

示例：

```cpp
float* d_A = nullptr;
cudaMalloc((void**)&d_A, n * sizeof(float));
```

这里的 `d_A` 指向的就是 device 端 global memory。

---

## 三、global memory 的典型用途

global memory 常用于存储：

1. 大规模输入数组。
2. 大规模输出数组。
3. 矩阵。
4. 图像。
5. 体数据。
6. 多个 kernel 之间共享的中间结果。
7. 需要 host-device 数据传输的数据。

例如，向量加法中：

```cpp
C[i] = A[i] + B[i];
```

数组 `A`、`B` 和 `C` 通常都存放在 global memory 中。

矩阵乘法中，矩阵 `A`、`B` 和 `C` 也通常先放在 global memory 中。基础版本直接访问 global memory，优化版本会将部分数据加载到 shared memory 中以减少重复读取。

---

## 四、global memory 的分配与释放

## 4.1 使用 `cudaMalloc` 分配

`cudaMalloc` 用于在 device 端分配 global memory。

函数形式：

```cpp
cudaError_t cudaMalloc(void** devPtr, size_t size);
```

示例：

```cpp
float* d_A = nullptr;
size_t bytes = n * sizeof(float);

cudaMalloc((void**)&d_A, bytes);
```

其中：

- `d_A` 是 device pointer。
- `bytes` 是分配的字节数。
- `cudaMalloc` 分配的是 device 端 global memory。

注意，`cudaMalloc` 的第二个参数是字节数，不是元素个数。

错误写法：

```cpp
cudaMalloc((void**)&d_A, n);
```

正确写法：

```cpp
cudaMalloc((void**)&d_A, n * sizeof(float));
```

---

## 4.2 使用 `cudaFree` 释放

使用 `cudaMalloc` 分配的 global memory 必须使用 `cudaFree` 释放。

示例：

```cpp
cudaFree(d_A);
```

完整示例：

```cpp
float* d_A = nullptr;
size_t bytes = n * sizeof(float);

cudaMalloc((void**)&d_A, bytes);

// 使用 d_A

cudaFree(d_A);
```

如果程序中分配了多个 device pointer，应逐一释放：

```cpp
cudaFree(d_A);
cudaFree(d_B);
cudaFree(d_C);
```

---

## 4.3 错误检查

正式 CUDA 程序中，建议对 `cudaMalloc` 和 `cudaFree` 进行错误检查。

```cpp
#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)
```

使用：

```cpp
CUDA_CHECK(cudaMalloc((void**)&d_A, bytes));
CUDA_CHECK(cudaFree(d_A));
```

这样可以及时发现显存不足、非法参数等问题。

## 五、host 与 global memory 的数据传输

host memory 和 device global memory 是两个不同的存储空间。普通 CPU 指针不能直接作为 GPU kernel 的输入，必须先将数据复制到 device memory。

---

## 5.1 host 到 device

使用：

```cpp
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
```

含义是：

```text
从 host pointer h_A 复制到 device pointer d_A
```

示例：

```cpp
float* h_A = static_cast<float*>(malloc(bytes));
float* d_A = nullptr;

cudaMalloc((void**)&d_A, bytes);
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
```

---

## 5.2 device 到 host

使用：

```cpp
cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost);
```

含义是：

```text
从 device pointer d_C 复制到 host pointer h_C
```

示例：

```cpp
cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost);
```

---

## 5.3 device 到 device

如果两个指针都在 device 端，可以使用：

```cpp
cudaMemcpy(d_B, d_A, bytes, cudaMemcpyDeviceToDevice);
```

---

## 5.4 常见 `cudaMemcpy` 方向

| 方向 | 写法 |
|---|---|
| host 到 device | `cudaMemcpyHostToDevice` |
| device 到 host | `cudaMemcpyDeviceToHost` |
| device 到 device | `cudaMemcpyDeviceToDevice` |
| host 到 host | `cudaMemcpyHostToHost` |

---

## 5.5 数据传输的性能问题

host-device 数据传输通常比 GPU 内部计算慢得多。优化 CUDA 程序时，应尽量减少不必要的数据传输。

低效做法：

```text
每执行一次小计算，就把数据从 device 复制回 host。
```

更好的做法：

```text
多个 kernel 连续在 GPU 上执行，中间结果保留在 global memory 中，最后一次性复制回 host。
```

示例：

```cpp
kernel1<<<grid, block>>>(d_A, d_B);
kernel2<<<grid, block>>>(d_B, d_C);
kernel3<<<grid, block>>>(d_C, d_D);

cudaMemcpy(h_D, d_D, bytes, cudaMemcpyDeviceToHost);
```

这样可以避免频繁 host-device 往返传输。

## 六、kernel 中访问 global memory

在 kernel 中，global memory 通常通过 device pointer 访问。

示例：

```cpp
__global__ void vectorAddKernel(const float* A, const float* B, float* C, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        C[idx] = A[idx] + B[idx];
    }
}
```

这里：

- `A` 指向 global memory。
- `B` 指向 global memory。
- `C` 指向 global memory。
- 每个线程读取 `A[idx]` 和 `B[idx]`，写入 `C[idx]`。

这种访问模式是全局存储器中最常见、也最容易优化的模式。

## 七、连续访问与合并访问

## 7.1 coalesced access 的含义

coalesced access，即合并访问，是 global memory 优化中最重要的概念之一。

它的核心思想是：

```text
连续线程访问连续内存地址。
```

例如，一个 warp 中的线程分别访问：

```text
A[0], A[1], A[2], A[3], ...
```

这种模式通常可以被硬件合并为更少的内存事务，从而更高效地利用 global memory 带宽。

典型代码：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;

if (idx < n) {
    C[idx] = A[idx] + B[idx];
}
```

这里相邻线程访问相邻的数组元素，通常可以形成较好的合并访问。

---

## 7.2 half-warp 与合并访问

教材中使用 half-warp 分析全局存储器访问。half-warp 是 16 个线程的集合。在较早 CUDA 架构中，硬件会以 half-warp 为单位分析线程访问的全局存储器地址是否连续、是否对齐。

如果 half-warp 中的线程访问连续地址，并且地址满足对齐要求，就可以形成合并访问。

例如，16 个线程访问 16 个连续 `float`：

```text
thread 0  -> A[0]
thread 1  -> A[1]
thread 2  -> A[2]
...
thread 15 -> A[15]
```

由于 `float` 为 4 字节，16 个 `float` 正好是 64 字节。如果起始地址也满足对齐要求，这种访问模式通常比较高效。

---

## 7.3 对齐访问

对齐访问 aligned access 指访问起始地址满足硬件要求。

例如，每个线程访问 32 bit 数据，也就是 4 字节数据。一个 half-warp 有 16 个线程，总访问范围是：

```text
16 × 4B = 64B
```

此时，如果第一个线程访问的地址是 64B 的整数倍，就更容易满足对齐访问要求。

一般优化原则是：

```text
线程访问的起始地址最好按照数据访问宽度和线程组大小对齐。
```

现代 GPU 对合并访问条件的限制已经比早期设备宽松，但连续、对齐访问仍然是优化 global memory 的基本原则。

---

## 八、非合并访问的性能问题

## 8.1 间隔访问

间隔访问 strided access 是常见的低效访问模式。

示例：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;
output[idx] = input[idx * stride];
```

如果 `stride` 较大，则相邻线程访问的地址相隔较远：

```text
thread 0 -> input[0]
thread 1 -> input[stride]
thread 2 -> input[2 * stride]
```

这种模式会降低 global memory 带宽利用率。

---

## 8.2 交叉访问

交叉访问指线程访问顺序与内存连续顺序不一致。

低效示例：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;
int j = (idx * 17) % n;

output[idx] = input[j];
```

这种访问模式不连续，也难以合并。

---

## 8.3 按列访问二维数组

二维数组在 C/C++ 中通常按行优先 row-major 顺序存储。

对于矩阵：

```cpp
A[row * width + col]
```

同一行相邻列元素在内存中连续：

```text
A[row][0], A[row][1], A[row][2], ...
```

但同一列相邻行元素在内存中相隔 `width` 个元素：

```text
A[0][col], A[1][col], A[2][col], ...
```

如果一个 warp 中的线程同时访问同一列，就可能形成间隔访问，导致 global memory 访问效率下降。

低效访问示例：

```cpp
int col = blockIdx.x * blockDim.x + threadIdx.x;
float value = A[col * width + fixed_col];
```

更理想的方式是让相邻线程访问同一行中的相邻列：

```cpp
int x = blockIdx.x * blockDim.x + threadIdx.x;
int y = blockIdx.y * blockDim.y + threadIdx.y;

float value = A[y * width + x];
```

## 九、计算能力差异

早期 CUDA 设备对 global memory 合并访问要求较严格。

在计算能力 1.0 和 1.1 的设备上，half-warp 访问需要满足更严格的顺序性和对齐条件。如果不满足，可能会被拆分为多次串行访问，性能下降明显。

在计算能力 1.2 及以上的设备上，限制有所放宽。只要线程访问落在合适的数据段中，即使不是完全顺序访问，硬件也可能以较少的内存事务完成访问。但如果访问跨越多个段，或者请求了大量无用数据，仍然会浪费带宽。

现代 CUDA 编程中，不需要死记早期架构的所有细节，但仍应遵循以下原则：

```text
连续线程访问连续地址。
访问起始地址尽量对齐。
避免大跨度 strided access。
避免结构体导致的非对齐访问。
尽量减少无效带宽浪费。
```

## 十、结构体数组的访问问题

## 10.1 AoS 布局

AoS 是 Array of Structures，即结构体数组。

示例：

```cpp
struct Vec3 {
    float x;
    float y;
    float z;
};

Vec3* data;
```

访问：

```cpp
output[idx] = data[idx].x * data[idx].x
            + data[idx].y * data[idx].y
            + data[idx].z * data[idx].z;
```

`Vec3` 的大小是 12 字节，而不是 4、8 或 16 字节。这可能导致访问对齐不理想，从而影响 global memory 访问效率。

---

## 10.2 使用 `__align__` 对齐结构体

可以使用 `__align__` 对结构体进行对齐。

示例：

```cpp
struct __align__(16) Vec3 {
    float x;
    float y;
    float z;
};
```

这样编译器会将结构体大小补齐到 16 字节，有助于满足对齐访问要求。

需要注意的是，这会增加内存占用，因为每个元素会多出填充空间。

---

## 10.3 SoA 布局

SoA 是 Structure of Arrays，即将结构体拆成多个连续数组。

原始 AoS：

```cpp
struct Vec3 {
    float x;
    float y;
    float z;
};

Vec3* data;
```

改为 SoA：

```cpp
float* x;
float* y;
float* z;
```

kernel 写法：

```cpp
__global__ void computeKernel(const float* x,
                              const float* y,
                              const float* z,
                              float* output,
                              int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        output[idx] = x[idx] * x[idx] + y[idx] * y[idx] + z[idx] * z[idx];
    }
}
```

SoA 的优点是：

```text
相邻线程访问 x[idx]、y[idx]、z[idx] 时，各自都是连续访问。
```

因此，它通常比 AoS 更适合 CUDA 中的大规模并行读取。

---

## 10.4 AoS 与 SoA 对比

| 布局 | 形式 | 优点 | 缺点 |
|---|---|---|---|
| AoS | `Vec3 data[n]` | 逻辑直观，单个对象数据集中 | 可能导致非对齐或非合并访问 |
| SoA | `x[n]`, `y[n]`, `z[n]` | 更利于连续线程连续访问 | 代码结构不如 AoS 直观 |

CUDA 中，如果每个线程都读取同一个字段的大量元素，SoA 通常更有利。

---

## 十一、借助 shared memory 调整访问

当 global memory 访问模式不理想时，可以借助 shared memory 做中间缓冲。

思路是：

```text
先让线程以连续方式从 global memory 读取数据到 shared memory；
再在 shared memory 中按照计算需要重新组织访问。
```

示例结构：

```cpp
__global__ void kernel(const float* input, float* output, int n) {
    __shared__ float tile[256];

    int tid = threadIdx.x;
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        tile[tid] = input[idx];
    }

    __syncthreads();

    if (idx < n) {
        output[idx] = tile[tid];
    }
}
```

这种思想在矩阵转置、图像处理和非规则访问优化中非常常见。

需要注意：

1. shared memory 本身容量有限。
2. shared memory 访问也可能发生 bank conflict。
3. 使用 shared memory 会增加代码复杂度和同步开销。
4. 只有当数据存在复用或访问重排需求时才值得使用。

## 十二、二维数组与线性存储

## 12.1 二维数组在 global memory 中的存储方式

global memory 本质上是线性存储器。二维数组通常按行优先方式展开为一维数组。

二维坐标：

```text
(row, col)
```

线性下标：

```cpp
int idx = row * width + col;
```

示例：

```cpp
__global__ void matrixAddKernel(const float* A,
                                const float* B,
                                float* C,
                                int width,
                                int height) {
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    int row = blockIdx.y * blockDim.y + threadIdx.y;

    if (col < width && row < height) {
        int idx = row * width + col;
        C[idx] = A[idx] + B[idx];
    }
}
```

这种写法中，如果 `threadIdx.x` 对应列方向，那么相邻线程访问相邻列，通常能形成较好的连续访问。

---

## 12.2 为什么优先让 threadIdx.x 对应列方向

在行优先存储中，同一行的相邻列地址连续。

因此推荐：

```cpp
int col = blockIdx.x * blockDim.x + threadIdx.x;
int row = blockIdx.y * blockDim.y + threadIdx.y;
```

而不是让 `threadIdx.x` 对应行方向。

较差写法：

```cpp
int row = blockIdx.x * blockDim.x + threadIdx.x;
int col = blockIdx.y * blockDim.y + threadIdx.y;
int idx = row * width + col;
```

在这种写法中，相邻 `threadIdx.x` 线程访问的是不同行的元素，地址间隔为 `width`，可能导致非合并访问。

---

## 十三、pitch memory

## 13.1 为什么需要 pitch memory

二维数组每一行的长度不一定天然满足硬件对齐要求。为了提高二维数据访问效率，CUDA 提供 `cudaMallocPitch` 分配带 pitch 的二维线性存储空间。

`pitch` 表示每一行实际占用的字节数，可能大于逻辑宽度对应的字节数。

例如，逻辑宽度是：

```text
width * sizeof(float)
```

但实际 pitch 可能被 CUDA 补齐为更适合对齐访问的字节数。

---

## 13.2 使用 `cudaMallocPitch`

函数形式：

```cpp
cudaError_t cudaMallocPitch(void** devPtr,
                            size_t* pitch,
                            size_t widthInBytes,
                            size_t height);
```

示例：

```cpp
float* d_A = nullptr;
size_t pitch = 0;

CUDA_CHECK(cudaMallocPitch((void**)&d_A,
                           &pitch,
                           width * sizeof(float),
                           height));
```

其中：

- `d_A` 是 device pointer。
- `pitch` 是每一行实际占用的字节数。
- `width * sizeof(float)` 是逻辑行宽字节数。
- `height` 是行数。

---

## 13.3 kernel 中访问 pitch memory

使用 pitch memory 时，不能简单写：

```cpp
idx = row * width + col;
```

因为每一行实际跨度不是 `width * sizeof(float)`，而是 `pitch` 字节。

正确方式：

```cpp
__global__ void kernel(float* data, size_t pitch, int width, int height) {
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    int row = blockIdx.y * blockDim.y + threadIdx.y;

    if (col < width && row < height) {
        char* rowPtr = (char*)data + row * pitch;
        float* rowData = (float*)rowPtr;

        rowData[col] = rowData[col] * 2.0f;
    }
}
```

这里必须先按字节偏移：

```cpp
(char*)data + row * pitch
```

再转换回对应数据类型：

```cpp
(float*)rowPtr
```

---

## 13.4 使用 `cudaMemcpy2D`

对于 pitch memory，host-device 复制通常使用 `cudaMemcpy2D`。

host 到 device：

```cpp
CUDA_CHECK(cudaMemcpy2D(d_A,
                        pitch,
                        h_A,
                        width * sizeof(float),
                        width * sizeof(float),
                        height,
                        cudaMemcpyHostToDevice));
```

device 到 host：

```cpp
CUDA_CHECK(cudaMemcpy2D(h_A,
                        width * sizeof(float),
                        d_A,
                        pitch,
                        width * sizeof(float),
                        height,
                        cudaMemcpyDeviceToHost));
```

参数含义：

```text
目标指针
目标 pitch
源指针
源 pitch
每行复制字节数
行数
复制方向
```

---

## 13.5 pitch memory 完整示例

```cpp
#include <cstdio>
#include <cstdlib>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void scale2DKernel(float* data, size_t pitch, int width, int height) {
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    int row = blockIdx.y * blockDim.y + threadIdx.y;

    if (col < width && row < height) {
        char* rowPtr = (char*)data + row * pitch;
        float* rowData = (float*)rowPtr;

        rowData[col] *= 2.0f;
    }
}

int main() {
    const int width = 1024;
    const int height = 512;

    const size_t rowBytes = width * sizeof(float);
    const size_t totalBytes = rowBytes * height;

    float* h_data = static_cast<float*>(malloc(totalBytes));

    if (h_data == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int i = 0; i < width * height; ++i) {
        h_data[i] = 1.0f;
    }

    float* d_data = nullptr;
    size_t pitch = 0;

    CUDA_CHECK(cudaMallocPitch((void**)&d_data, &pitch, rowBytes, height));

    CUDA_CHECK(cudaMemcpy2D(d_data,
                            pitch,
                            h_data,
                            rowBytes,
                            rowBytes,
                            height,
                            cudaMemcpyHostToDevice));

    dim3 block(16, 16);
    dim3 grid((width + block.x - 1) / block.x,
              (height + block.y - 1) / block.y);

    scale2DKernel<<<grid, block>>>(d_data, pitch, width, height);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy2D(h_data,
                            rowBytes,
                            d_data,
                            pitch,
                            rowBytes,
                            height,
                            cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < width * height; ++i) {
        if (h_data[i] != 2.0f) {
            correct = false;
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaFree(d_data));
    free(h_data);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

编译：

```bash
nvcc -O2 pitch_example.cu -o pitch_example
```

运行：

```bash
./pitch_example
```

预期输出：

```text
Result: PASS
```

## 十四、global memory 常见优化方法

## 14.1 减少 global memory 访问次数

global memory 访问延迟高，因此应避免重复访问。

低效示例：

```cpp
float a = input[idx];
float b = input[idx];
float c = input[idx];
output[idx] = a + b + c;
```

更好写法：

```cpp
float value = input[idx];
output[idx] = value + value + value;
```

---

## 14.2 使用 register 保存临时值

线程内部多次使用的标量应保存在局部变量中，让编译器尽量放入 register。

```cpp
float value = input[idx];
float result = value * value + 2.0f * value + 1.0f;
output[idx] = result;
```

---

## 14.3 使用 shared memory 缓存可复用数据

如果多个线程会重复读取相邻数据，可以考虑 shared memory。

适用场景：

1. 矩阵乘法。
2. 图像卷积。
3. stencil 计算。
4. block 内归约。
5. 矩阵转置。

---

## 14.4 保证连续线程访问连续地址

推荐写法：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;
output[idx] = input[idx];
```

避免：

```cpp
output[idx] = input[idx * stride];
```

---

## 14.5 让二维数组按行访问

推荐：

```cpp
int col = blockIdx.x * blockDim.x + threadIdx.x;
int row = blockIdx.y * blockDim.y + threadIdx.y;
int idx = row * width + col;
```

这样相邻 `threadIdx.x` 对应相邻内存地址。

---

## 14.6 使用 SoA 代替 AoS

如果结构体数组导致访问不连续或不对齐，可以将：

```cpp
struct Vec3 {
    float x;
    float y;
    float z;
};

Vec3* data;
```

改为：

```cpp
float* x;
float* y;
float* z;
```

这样相邻线程访问 `x[idx]`、`y[idx]`、`z[idx]` 时都是连续访问。

---

## 14.7 使用 `__align__` 对齐结构体

如果必须使用结构体数组，可以考虑：

```cpp
struct __align__(16) Vec3 {
    float x;
    float y;
    float z;
};
```

这样结构体大小会补齐到 16 字节，有利于对齐访问。

---

## 14.8 使用 pitch memory 优化二维数据

对于二维数组，可以使用：

```cpp
cudaMallocPitch
cudaMemcpy2D
```

让每一行满足更合适的对齐要求。

---

## 14.9 增加足够的并行度隐藏访存延迟

当 global memory 访问不可避免时，可以通过足够多的活跃线程隐藏访存延迟。

常见方法：

1. 启动足够多的 block。
2. block size 选择 128、256、512 等 32 的倍数。
3. 避免每个线程使用过多寄存器导致 occupancy 下降。
4. 在两次 global memory 访问之间安排足够计算。

需要注意：增加线程数量不能解决所有问题。如果访问模式本身很差，仍然需要优化访存模式。

---

## 十五、global memory 使用示例：向量加法

向量加法是最典型的 global memory 访问示例。

```cpp
#include <cstdio>
#include <cstdlib>
#include <cmath>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void vectorAddKernel(const float* A, const float* B, float* C, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    if (idx < n) {
        C[idx] = A[idx] + B[idx];
    }
}

int main() {
    const int n = 1 << 20;
    const size_t bytes = n * sizeof(float);

    float* h_A = static_cast<float*>(malloc(bytes));
    float* h_B = static_cast<float*>(malloc(bytes));
    float* h_C = static_cast<float*>(malloc(bytes));

    if (h_A == nullptr || h_B == nullptr || h_C == nullptr) {
        fprintf(stderr, "Host allocation failed.\n");
        return EXIT_FAILURE;
    }

    for (int i = 0; i < n; ++i) {
        h_A[i] = static_cast<float>(i);
        h_B[i] = static_cast<float>(2 * i);
    }

    float* d_A = nullptr;
    float* d_B = nullptr;
    float* d_C = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_A, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_B, bytes));
    CUDA_CHECK(cudaMalloc((void**)&d_C, bytes));

    CUDA_CHECK(cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice));
    CUDA_CHECK(cudaMemcpy(d_B, h_B, bytes, cudaMemcpyHostToDevice));

    dim3 block(256);
    dim3 grid((n + block.x - 1) / block.x);

    vectorAddKernel<<<grid, block>>>(d_A, d_B, d_C, n);
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost));

    bool correct = true;
    for (int i = 0; i < n; ++i) {
        float expected = h_A[i] + h_B[i];

        if (fabs(h_C[i] - expected) > 1e-5f) {
            correct = false;
            break;
        }
    }

    printf("Result: %s\n", correct ? "PASS" : "FAIL");

    CUDA_CHECK(cudaFree(d_A));
    CUDA_CHECK(cudaFree(d_B));
    CUDA_CHECK(cudaFree(d_C));

    free(h_A);
    free(h_B);
    free(h_C);

    return correct ? EXIT_SUCCESS : EXIT_FAILURE;
}
```

这个程序的 global memory 访问模式较好，因为相邻线程访问相邻元素：

```cpp
A[idx]
B[idx]
C[idx]
```

## 十六、常见错误

## 16.1 `cudaMalloc` 字节数错误

错误：

```cpp
cudaMalloc((void**)&d_A, n);
```

正确：

```cpp
cudaMalloc((void**)&d_A, n * sizeof(float));
```

---

## 16.2 `cudaMemcpy` 方向错误

错误：

```cpp
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyDeviceToHost);
```

正确：

```cpp
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
```

---

## 16.3 host pointer 与 device pointer 混用

错误：

```cpp
kernel<<<grid, block>>>(h_A, h_B, h_C, n);
```

正确：

```cpp
kernel<<<grid, block>>>(d_A, d_B, d_C, n);
```

kernel 在 GPU 上执行，不能直接使用普通 host pointer。

---

## 16.4 忘记边界判断

错误：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;
C[idx] = A[idx] + B[idx];
```

正确：

```cpp
int idx = blockIdx.x * blockDim.x + threadIdx.x;

if (idx < n) {
    C[idx] = A[idx] + B[idx];
}
```

因为启动的线程总数通常会大于实际数据规模。

---

## 16.5 忘记释放 global memory

错误：

```cpp
cudaMalloc((void**)&d_A, bytes);
// 没有 cudaFree(d_A)
```

正确：

```cpp
cudaFree(d_A);
```

---

## 16.6 误认为 `cudaMalloc` 分配的是 host memory

`cudaMalloc` 分配的是 device 端 global memory，不是 CPU 主存。

host 端不能像访问普通数组一样直接访问 `d_A`：

```cpp
d_A[0] = 1.0f;  // 错误
```

正确做法是：

```cpp
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
```

或在 kernel 中访问：

```cpp
kernel<<<grid, block>>>(d_A);
```

---

## 16.7 pitch memory 下标计算错误

错误：

```cpp
int idx = row * width + col;
data[idx] = value;
```

如果使用 `cudaMallocPitch`，每行实际跨度是 `pitch` 字节，而不是 `width * sizeof(float)`。

正确：

```cpp
char* rowPtr = (char*)data + row * pitch;
float* rowData = (float*)rowPtr;
rowData[col] = value;
```

---

## 16.8 结构体大小导致非对齐访问

错误或低效布局：

```cpp
struct Vec3 {
    float x;
    float y;
    float z;
};
```

`Vec3` 大小为 12 字节，可能不利于对齐访问。

可以考虑：

```cpp
struct __align__(16) Vec3 {
    float x;
    float y;
    float z;
};
```

或改为 SoA：

```cpp
float* x;
float* y;
float* z;
```

## 十七、回答模板

## 17.1 解释 global memory

```text
global memory 是 GPU 板载显存中的全局存储器，容量较大，所有 CUDA 线程都可以访问，host 端也可以通过 cudaMalloc、cudaMemcpy 和 cudaFree 等 Runtime API 进行管理。它通常用于存储大规模输入数组、输出数组和多个 kernel 之间共享的中间结果。global memory 的访问延迟较高，因此使用时应尽量保证连续线程访问连续地址，以形成合并访问，提高带宽利用率。
```

---

## 17.2 解释为什么 global memory 慢

```text
global memory 位于 GPU 片外显存中，距离执行单元较远，因此访问延迟明显高于 register 和 shared memory。虽然 global memory 的理论带宽较高，但只有在访问模式合理、能够形成合并访问时，才能较好地利用这些带宽。如果线程访问地址分散、间隔过大或不对齐，就会产生更多内存事务，导致性能下降。
```

---

## 17.3 解释 coalesced access

```text
coalesced access 指合并访问，核心思想是让同一个 warp 或 half-warp 中的连续线程访问 global memory 中的连续地址。这样硬件可以把多个线程的访存请求合并成更少的内存事务，从而提高显存带宽利用率。典型写法是 int idx = blockIdx.x * blockDim.x + threadIdx.x，然后让线程访问 A[idx]、B[idx] 和 C[idx]。
```

---

## 17.4 解释 AoS 和 SoA

```text
AoS 是结构体数组，例如 Vec3 data[n]，每个元素中包含 x、y、z。它逻辑直观，但结构体大小如果不是 4、8 或 16 字节的友好对齐大小，可能导致全局存储器访问效率下降。SoA 是把结构体拆成多个数组，例如 x[n]、y[n]、z[n]，这样相邻线程访问同一字段时地址连续，通常更适合 CUDA 中的合并访问。
```

---

## 17.5 解释 `cudaMallocPitch`

```text
cudaMallocPitch 用于分配二维线性显存。它会为每一行加入必要的填充，使每一行的起始地址更容易满足对齐要求。函数会返回 pitch，表示每一行实际占用的字节数。使用 pitch memory 时，kernel 中不能简单使用 row * width + col 计算地址，而应使用 (char*)ptr + row * pitch 找到对应行，再转换为具体数据类型进行访问。
```

---

## 17.6 解释如何优化 global memory

```text
优化 global memory 的核心是减少访问次数并提高访问效率。首先应避免不必要的 host-device 数据传输和重复读取；其次应让连续线程访问连续地址，形成合并访问；对于二维数组，应让 threadIdx.x 对应列方向，使相邻线程访问同一行的相邻元素；对于结构体数组，可以使用 __align__ 对齐或改为 SoA 布局；当数据存在复用时，可以使用 shared memory 缓存局部数据。
```

## 十九、小结

global memory 是 CUDA 程序中最常用的大容量 device memory。它位于 GPU 板载显存中，容量大、访问范围广，可以被所有线程访问，也可以由 host 端通过 CUDA Runtime API 进行管理。常见的管理函数包括 `cudaMalloc`、`cudaMemcpy` 和 `cudaFree`，二维数据还可以使用 `cudaMallocPitch` 和 `cudaMemcpy2D` 来满足更好的对齐要求。

global memory 的主要问题是访问延迟高。虽然显存带宽较大，但只有在线程访问模式合理时才能有效利用。优化 global memory 的关键是尽量让连续线程访问连续地址，形成 coalesced access；避免间隔访问、交叉访问和非对齐访问；对于二维数组，应优先让相邻线程访问同一行中的相邻元素；对于结构体数组，应注意结构体大小和对齐问题，必要时使用 `__align__` 或改为 SoA 布局。

本节的核心结论是：global memory 应用于大规模数据存储，但不能随意访问。CUDA 程序优化时，必须重点检查 global memory 的访问次数、访问连续性、对齐情况和 host-device 数据传输开销。
