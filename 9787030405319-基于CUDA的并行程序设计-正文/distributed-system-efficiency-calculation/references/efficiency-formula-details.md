# 系统运行效能公式详解

## 公式推导背景
该效能指标衡量“节点数 × 运行时间”这一资源消耗量相对于理想线性扩展的效率。理想情况下，若系统完全线性扩展，则 n 节点运行时间应为 T_2 × (2/n)，此时效能为100%。

## 变量说明
- **T_2**: 2节点配置下的系统总运行时间（秒）
- **T_n**: n节点配置下的系统总运行时间（秒）
- **n**: 实际使用的节点数量，整数且 n ≥ 2

## 约束条件
- 仅适用于同一问题规模（如 N_max=7 固定）下的比较。
- 不适用于异构硬件环境（如CPU型号、内存带宽不同）。
- 假设2节点为系统可运行的最小配置。

## 示例（来自表9.11）
- 2节点：T_2 = 100s → 效能 = (2×100)/(2×100) = 1.0 → 100%
- 3节点：T_3 = 70s → 效能 = (2×100)/(3×70) ≈ 0.952 → 95.2%

## 常见误用
- 在问题规模变化时使用（如N_max从7变为10）→ 结果不可比。
- 在混合CPU/GPU或不同代际硬件上使用 → 违反同构假设。
- 使用n=1 → 公式不适用，因2节点为最小基准。
