# CUDA性能分析三大核心指标

在完成CUDA程序性能数据采集后，执行以下三步分析流程，以系统化评估性能瓶颈：

## 1. 评估带宽利用率（Bandwidth Utilization）
- 计算程序实际带宽与设备理论带宽的比值。
- 考察范围包括主机与设备间的PCI-E带宽以及设备内部存储器带宽。
- 若该比率较低，说明数据传输是性能瓶颈，应优化内存访问模式或减少不必要的数据传输。

## 2. 评估设备利用率（Device Utilization）
- 计算每个SM上实际活动warp数与该SM支持的最大活动warp数的比值。
- 根据GPU的计算能力（如1.0/1.1、1.2/1.3、2.0/2.1等）确定最大warp容量。
- 若利用率过低，说明无法有效隐藏访存延迟，应增加并行度或减少分支发散；若已足够高，则进一步提升对性能帮助有限。

## 3. 评估平均指令吞吐量（Average Instruction Throughput）
- 计算内核执行期间所有SM指令吞吐量的平均值。
- 高吞吐量要求各SM负载均衡且每SM执行效率高。
- 若吞吐量偏低，检查是否存在负载不均、分支发散或低效指令使用。

综合三项指标结果，判断主要瓶颈所在，并优先优化影响最大的维度。
