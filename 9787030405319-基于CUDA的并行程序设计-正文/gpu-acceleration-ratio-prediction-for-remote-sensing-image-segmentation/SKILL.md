# GPU加速比预测（遥感图像水平集分割）

## 适用场景
- 使用NVIDIA GPU（如GTS 450）运行CUDA 4.2+环境下的水平集分割算法
- 处理高分辨率遥感图像（图像尺寸 ≥ 5548×11036，即约58M像素以上）
- 需要根据图像尺寸和分块大小预估GPU加速效果以优化参数配置

## 执行步骤
1. **确认前提条件**：
   - 确保设备为NVIDIA GPU（如GeForce GTS 450，1GB显存，48KB共享内存）
   - CUDA版本 ≥ 4.2，计算能力 ≥ 2.1
   - 图像具有高度并行性（如遥感图像水平集分割）

2. **评估图像尺寸影响**：
   - 若图像总像素 ≥ 100M（如104M或233M），预期基础加速比约为16.5–16.7倍
   - 小于58M像素的图像可能无法获得显著加速（<15倍）

3. **调整分块大小（PieceSize）**：
   - 在图像尺寸固定（尤其是≥100M像素）时，增大PieceSize可显著提升加速比
   - 推荐尝试PieceSize=500（若不超过GPU共享内存限制）
   - 注意：分块大小受限于SharedMemorySize，不可超出硬件约束

4. **预测加速比范围**：
   - 图像 ≥ 100M像素 且 PieceSize ≥ 500 → 预期加速比 25–30倍
   - 其他大图像（≥58M）但小分块（如PieceSize=100）→ 预期加速比 15–17倍

5. **验证与调优**：
   - 实际运行后记录GPU时间，计算加速比 = CPU串行时间 / GPU并行时间
   - 若加速比低于预期，检查数据传输开销或核函数启动频率是否过高

> **注意**：本技能不适用于小图像或低并行度任务，此类场景可能无显著加速甚至出现性能下降。
