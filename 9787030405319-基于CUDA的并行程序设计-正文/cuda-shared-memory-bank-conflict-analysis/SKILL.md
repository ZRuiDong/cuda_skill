# CUDA共享存储器Bank冲突分析

## 何时使用
当满足以下条件时应用本技能：
- 使用共享存储器进行线程间通信
- GPU计算能力为1.x或2.x
- 多个线程同时访问共享存储器
- 需要优化内存带宽利用率

## 执行步骤
1. **确定GPU计算能力**：
   - 1.x：16个bank，half-warp大小为16
   - 2.x：32个bank，warp大小为32

2. **判断访问模式**：
   - 相邻4字节数据分配至不同bank（如data[0]→bank0, data[1]→bank1, ..., data[15]→bank15, data[16]→bank0）

3. **检测冲突条件**：
   - 计算能力1.x：同一half-warp中多个线程访问同一bank的不同地址 → 冲突
   - 计算能力2.x：同一warp中多个线程访问同一bank的不同地址 → 冲突

4. **识别无冲突特例**：
   - 所有线程访问同一bank的同一4字节地址 → 硬件广播，无冲突

5. **评估性能影响**：
   - bank冲突导致访存请求串行化，有效带宽降至原来的1/n（n为冲突路数）

> 注意：本技能仅适用于共享存储器，不适用于全局或常量存储器。
