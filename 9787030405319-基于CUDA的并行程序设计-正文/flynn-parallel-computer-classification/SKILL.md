# Flynn并行计算机分类法

## 何时使用
- 需要对给定的计算机体系结构进行并行性分类
- 设计或评估并行计算系统时确定其基本架构类型
- 为并行编程模型（如OpenMP、CUDA）选择合适的硬件抽象基础
- 教学或技术文档中解释并行计算机的基本分类框架

## 执行步骤

1. **判断指令流数量**：
   - 若系统在任一时刻仅执行一条指令流 → `单指令流（Single Instruction）`
   - 若多个处理单元可同时执行不同指令 → `多指令流（Multiple Instructions）`

2. **判断数据流数量**：
   - 若所有操作作用于同一数据项 → `单数据流（Single Data）`
   - 若多个处理单元操作各自独立的数据 → `多数据流（Multiple Data）`

3. **组合分类结果**：
   - **SISD**（单指令流单数据流）：传统串行处理器，可能含流水线，但无真正并行
   - **SIMD**（单指令流多数据流）：单一控制器同步多个处理单元，适用于图像处理、向量化计算等规则并行任务
   - **MISD**（多指令流单数据流）：极少见，如某些脉动阵列；同一数据被多个不同指令依次处理
   - **MIMD**（多指令流多数据流）：主流并行架构，各处理器独立运行，支持共享内存（如多核CPU）或消息传递（如集群）

4. **注意事项**：
   - SISD虽属串行，但可通过流水线实现操作级并行，仍归为SISD
   - SIMD要求任务具有高度规则性和数据并行性
   - MIMD灵活性最高，适用于通用并行计算场景
   - MISD在现实中几乎无实际商用系统对应，通常仅作理论分类

> ⚠️ 此分类法不适用于非冯·诺依曼架构（如神经形态计算、量子计算）。
