# 确定性任务调度模型

## 何时使用
- 所有任务之间的**优先关系**在调度前已完全确定。
- 每个任务的**执行时间**已知且固定。
- 目标平台为**MIMD系统**，需进行**静态任务调度**以支持**并行算法实现**。

## 如何执行
1. **构建任务图（Task Graph）**：
   - 使用有向无环图（DAG）表示任务及其依赖关系。
   - 节点表示任务，并标注其执行时间。
   - 有向边表示任务间的优先约束（前驱必须在后继之前完成）。

2. **生成调度方案**：
   - 将任务分配到多个处理机上，满足优先关系约束。
   - 优化目标：最小化整体完成时间（makespan）。

3. **可视化调度结果**：
   - 使用**甘特图（Gantt Chart）** 展示调度结果：
     - 横轴为时间，纵轴为处理机。
     - 条形块表示任务在特定处理机上的执行时间段。
     - 确保图中体现任务依赖与资源分配。

4. **识别潜在挑战**：
   - 即使输入信息完备，仍需警惕**存储冲突**（多任务争用内存）和**中断干扰**（如操作系统事件）对实际执行的影响。

> 注意：本技能仅适用于静态、信息完备的调度场景。若任务依赖或执行时间在运行时动态变化，请使用动态调度方法。
