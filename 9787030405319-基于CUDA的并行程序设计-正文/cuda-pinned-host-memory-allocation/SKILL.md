# CUDA分页锁定主机内存分配

## 何时使用
当满足以下条件时使用本技能：
- 需要高频或异步地在主机（CPU）与设备（GPU）之间传输数据
- 系统有足够物理内存
- 数据路径对性能敏感且属于关键路径

**注意**：不应大量使用，以免降低系统整体性能；仅用于关键数据交换缓冲区。

## 如何执行
1. 调用 `cudaError_t cudaMallocHost(void **hostPtr, size_t size)` 分配指定字节数的分页锁定主机内存。
2. 检查返回值：
   - 若为 `cudaSuccess`，则 `*hostPtr` 指向已分配的分页锁定内存
   - 若为 `cudaErrorMemoryAllocation`，表示分配失败（如内存不足）
3. 使用该内存进行主机-设备数据传输：
   - 可被GPU直接访问，无需CPU中转
   - `cudaMemcpy*` 系列函数对此类内存自动优化
   - **仅此类内存支持 `cudaMemcpyAsync` 等异步操作**
4. 使用完毕后，必须调用 `cudaFreeHost(*hostPtr)` 释放内存。

## 关键约束
- 传入异步函数的指针若未通过 `cudaMallocHost` 分配，将导致运行时错误
- 过度分配会减少系统可用于分页的内存，影响整体系统性能
- 仅用于频繁交换数据的缓冲区，避免用于一次性或低频用途
