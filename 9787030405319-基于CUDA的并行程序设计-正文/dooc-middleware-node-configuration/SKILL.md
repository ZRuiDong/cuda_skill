# D0oC中间件节点配置

## 何时使用
- 节点已加入D0oC集群
- 任务涉及CUDA核外计算
- 需明确节点类型（计算、数据或混合）并部署对应中间件组件

## 如何执行
1. **确定节点角色**：
   - **计算节点**：部署1个本地任务管理器、≥1个执行单元（每单元占1 CPU线程，可调用GPU）、1个数据服务模块
   - **数据节点**：部署1个数据服务模块 + 1个外存读写辅助模块
   - **混合节点**（兼具计算与数据功能）：部署1个本地任务管理器、≥1个执行单元、1个数据服务模块（不重复）、1个外存读写辅助模块
2. **设置全局任务管理器**：仅在主程序所在节点（如node1）部署，用于接收外部任务或内嵌流程代码
3. **连接规则**：
   - 执行单元连接本地任务管理器和数据服务模块，但彼此独立
   - 数据服务模块之间相互通信，无主从关系
   - 本地任务管理器仅与全局任务管理器通信，不互相连接

> 注意：纯数据节点不得配置本地任务管理器或执行单元；全局任务管理器全局唯一。
