# 3.6 HelloWorld CUDA 编程实例

## 一、本节作用

本节用于整理 CUDA 入门阶段最基础的 HelloWorld 编程实例。它的作用不是追求复杂计算，而是帮助学习者理解一个 CUDA 程序从编写、编译到执行的完整过程。

通过本节内容，应能够掌握以下内容：

1. CUDA 程序的最小结构。
2. `.cu` 文件的基本写法。
3. `__global__` kernel 函数的定义方式。
4. kernel 的启动语法。
5. host 端与 device 端之间的关系。
6. `threadIdx`、`blockIdx` 等线程编号的基本使用。
7. `nvcc` 编译 CUDA 程序的方法。
8. CUDA 入门程序中常见错误的排查方法。

本节通常在学习完 CUDA 编程模型之后使用。它是从“理解概念”过渡到“写出可运行 CUDA 程序”的第一个实践文件。

---

## 二、CUDA HelloWorld 的意义

普通 C/C++ 的 HelloWorld 程序通常只需要在 CPU 上调用 `printf` 输出字符串。

CUDA 的 HelloWorld 程序虽然也可以输出字符串，但它的真正意义在于展示 CPU 与 GPU 的协同执行过程：

1. CPU 端负责启动程序。
2. CPU 端调用 kernel。
3. GPU 端执行 kernel。
4. GPU 端的多个线程可以并行执行同一份代码。
5. CPU 端需要在适当位置等待 GPU 执行完成。

因此，CUDA HelloWorld 的重点不是输出内容本身，而是理解：

```text
host code 启动 device code
```

也就是：

```text
CPU 调用 GPU kernel
```

## 三、最小 CUDA 程序结构

一个最小 CUDA 程序通常包括：

1. 头文件。
2. kernel 函数。
3. `main` 函数。
4. kernel 启动语句。
5. 同步与错误检查。

最小结构如下：

```cpp
#include <cstdio>
#include <cuda_runtime.h>

__global__ void helloKernel() {
    printf("Hello from GPU!\n");
}

int main() {
    printf("Hello from CPU!\n");

    helloKernel<<<1, 1>>>();

    cudaDeviceSynchronize();

    return 0;
}
```

其中：

- `helloKernel` 是在 GPU 上执行的 kernel。
- `<<<1, 1>>>` 表示启动 1 个 block，每个 block 中有 1 个 thread。
- `cudaDeviceSynchronize()` 用于等待 GPU 端 kernel 执行完成。
- 如果没有 `cudaDeviceSynchronize()`，GPU 端的 `printf` 输出可能来不及显示，程序就已经结束。

## 五、线程编号输出示例

最小 HelloWorld 只启动一个线程，无法体现 CUDA 的并行线程结构。为了观察多个线程的执行，可以让每个线程输出自己的编号。

---

### 1. 代码

文件名建议使用：

```text
hello_thread_index.cu
```

代码如下：

```cpp
#include <cstdio>
#include <cuda_runtime.h>

__global__ void helloThreadKernel() {
    int global_id = blockIdx.x * blockDim.x + threadIdx.x;

    printf("Hello from GPU: blockIdx.x = %d, threadIdx.x = %d, global_id = %d\n",
           blockIdx.x, threadIdx.x, global_id);
}

int main() {
    printf("Hello from CPU!\n");

    dim3 block(4);
    dim3 grid(2);

    helloThreadKernel<<<grid, block>>>();

    cudaDeviceSynchronize();

    return 0;
}
```

---

### 2. 线程数量说明

上述程序中：

```cpp
dim3 block(4);
dim3 grid(2);
```

表示：

```text
每个 block 中有 4 个 thread
一共有 2 个 block
总线程数 = 2 × 4 = 8
```

每个线程都会执行一次 `helloThreadKernel` 中的代码。

---

### 3. 全局线程编号

代码中：

```cpp
int global_id = blockIdx.x * blockDim.x + threadIdx.x;
```

表示当前线程在整个 grid 中的一维全局编号。

例如：

| blockIdx.x | threadIdx.x | global_id |
|---|---|---|
| 0 | 0 | 0 |
| 0 | 1 | 1 |
| 0 | 2 | 2 |
| 0 | 3 | 3 |
| 1 | 0 | 4 |
| 1 | 1 | 5 |
| 1 | 2 | 6 |
| 1 | 3 | 7 |

---

### 4. 输出顺序说明

GPU 上多个线程是并行执行的，因此输出顺序不一定固定。

可能输出如下：

```text
Hello from CPU!
Hello from GPU: blockIdx.x = 0, threadIdx.x = 0, global_id = 0
Hello from GPU: blockIdx.x = 0, threadIdx.x = 1, global_id = 1
Hello from GPU: blockIdx.x = 1, threadIdx.x = 0, global_id = 4
Hello from GPU: blockIdx.x = 0, threadIdx.x = 2, global_id = 2
...
```

不能假设 GPU 线程会严格按照 `global_id` 从小到大输出。

## 六、带错误检查的 HelloWorld 示例

入门示例可以很短，但正式编写 CUDA 程序时，应加入错误检查。这样可以及时发现 kernel 启动失败、设备不可用、非法内存访问等问题。

---

### 1. 代码

文件名建议使用：

```text
hello_world_checked.cu
```

代码如下：

```cpp
#include <cstdio>
#include <cstdlib>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void helloKernel() {
    int global_id = blockIdx.x * blockDim.x + threadIdx.x;

    printf("Hello from GPU thread %d\n", global_id);
}

int main() {
    printf("Hello from CPU!\n");

    dim3 block(8);
    dim3 grid(2);

    helloKernel<<<grid, block>>>();

    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    return EXIT_SUCCESS;
}
```

---

### 2. 编译命令

```bash
nvcc -O2 hello_world_checked.cu -o hello_world_checked
```

---

### 3. 运行命令

```bash
./hello_world_checked
```

---

### 4. 说明

这里使用：

```cpp
CUDA_CHECK(cudaGetLastError());
```

检查 kernel 启动是否发生错误。

使用：

```cpp
CUDA_CHECK(cudaDeviceSynchronize());
```

等待 kernel 完成，同时捕获 kernel 执行期间可能发生的错误。

在后续 CUDA 程序中，建议保留这种错误检查方式。

## 七、教材风格的 GPU 字符串复制示例

教材中的 HelloWorld 示例不仅展示 GPU 输出，还演示了一个更完整的 CUDA 调用流程：host 端准备字符串，将字符串复制到 device 端，在 GPU 端执行 kernel，再把结果复制回 host 端。

这个示例虽然计算量很小，只启动一个线程，但它能直观展示 CUDA 程序的基本流程：

```text
host 字符串
    ↓ cudaMemcpyHostToDevice
device 字符串
    ↓ kernel 处理
device 结果
    ↓ cudaMemcpyDeviceToHost
host 输出结果
```

---

### 1. 单文件现代写法

文件名建议使用：

```text
gpu_string_copy.cu
```

代码如下：

```cpp
#include <cstdio>
#include <cstdlib>
#include <cstring>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

__global__ void copyStringKernel(const char* src, char* dst) {
    int i = 0;

    while (src[i] != '\0') {
        dst[i] = src[i];
        ++i;
    }

    dst[i] = '\0';
}

int main() {
    const char* h_src = "HelloWorld from CUDA";
    const int size = static_cast<int>(strlen(h_src)) + 1;

    char* h_dst = static_cast<char*>(malloc(size));

    if (h_dst == nullptr) {
        fprintf(stderr, "Host memory allocation failed.\n");
        return EXIT_FAILURE;
    }

    char* d_src = nullptr;
    char* d_dst = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_src, size));
    CUDA_CHECK(cudaMalloc((void**)&d_dst, size));

    CUDA_CHECK(cudaMemcpy(d_src, h_src, size, cudaMemcpyHostToDevice));

    copyStringKernel<<<1, 1>>>(d_src, d_dst);

    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_dst, d_dst, size, cudaMemcpyDeviceToHost));

    printf("%s\n", h_dst);

    CUDA_CHECK(cudaFree(d_src));
    CUDA_CHECK(cudaFree(d_dst));

    free(h_dst);

    return EXIT_SUCCESS;
}
```

---

### 2. 编译命令

```bash
nvcc -O2 gpu_string_copy.cu -o gpu_string_copy
```

---

### 3. 运行命令

```bash
./gpu_string_copy
```

---

### 4. 预期输出

```text
HelloWorld from CUDA
```

---

### 5. 示例说明

这个示例中，host 端字符串是：

```cpp
const char* h_src = "HelloWorld from CUDA";
```

device 端指针是：

```cpp
char* d_src = nullptr;
char* d_dst = nullptr;
```

通过：

```cpp
cudaMalloc
```

在 GPU 显存中分配空间。

通过：

```cpp
cudaMemcpy(d_src, h_src, size, cudaMemcpyHostToDevice);
```

将字符串从 CPU 主存复制到 GPU 显存。

通过：

```cpp
copyStringKernel<<<1, 1>>>(d_src, d_dst);
```

启动一个 block、一个 thread，在 GPU 上复制字符串。

通过：

```cpp
cudaMemcpy(h_dst, d_dst, size, cudaMemcpyDeviceToHost);
```

将 GPU 端结果复制回 CPU 主存并输出。

## 八、设备查询与初始化示例

在一些教材风格的 CUDA 程序中，会先检查当前机器是否存在支持 CUDA 的 GPU。可以使用 `cudaGetDeviceCount` 和 `cudaGetDeviceProperties` 查询设备信息。

---

### 1. 代码

```cpp
#include <cstdio>
#include <cuda_runtime.h>

bool initCUDA() {
    int count = 0;
    cudaError_t err = cudaGetDeviceCount(&count);

    if (err != cudaSuccess) {
        fprintf(stderr, "cudaGetDeviceCount failed: %s\n", cudaGetErrorString(err));
        return false;
    }

    if (count == 0) {
        fprintf(stderr, "There is no CUDA device.\n");
        return false;
    }

    for (int i = 0; i < count; ++i) {
        cudaDeviceProp prop;
        if (cudaGetDeviceProperties(&prop, i) == cudaSuccess) {
            printf("Device %d: %s\n", i, prop.name);
            printf("Compute capability: %d.%d\n", prop.major, prop.minor);

            cudaSetDevice(i);
            return true;
        }
    }

    fprintf(stderr, "No suitable CUDA device found.\n");
    return false;
}
```

---

### 2. 使用方式

可以在 `main` 函数开头调用：

```cpp
if (!initCUDA()) {
    return EXIT_FAILURE;
}
```

这样可以在程序开始时确认 CUDA 设备可用。

---

### 3. 说明

现代 CUDA 程序中，如果只是写简单作业或实验，通常不一定要单独封装 `initCUDA()`。但在较完整的工程中，设备查询和设备选择仍然很有用。

## 九、完整综合示例：设备检查 + GPU 字符串复制

下面给出一个更完整的 HelloWorld 示例，包含设备检查、显存分配、数据传输、kernel 启动、结果回传和资源释放。

文件名建议使用：

```text
hello_world_full.cu
```

代码如下：

```cpp
#include <cstdio>
#include <cstdlib>
#include <cstring>
#include <cuda_runtime.h>

#define CUDA_CHECK(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            fprintf(stderr, "CUDA error at %s:%d: %s\n",                   \
                    __FILE__, __LINE__, cudaGetErrorString(err));           \
            exit(EXIT_FAILURE);                                             \
        }                                                                  \
    } while (0)

bool initCUDA() {
    int count = 0;
    CUDA_CHECK(cudaGetDeviceCount(&count));

    if (count == 0) {
        fprintf(stderr, "There is no CUDA device.\n");
        return false;
    }

    int selectedDevice = -1;

    for (int i = 0; i < count; ++i) {
        cudaDeviceProp prop;
        CUDA_CHECK(cudaGetDeviceProperties(&prop, i));

        printf("Device %d: %s\n", i, prop.name);
        printf("Compute capability: %d.%d\n", prop.major, prop.minor);

        if (selectedDevice == -1) {
            selectedDevice = i;
        }
    }

    CUDA_CHECK(cudaSetDevice(selectedDevice));

    printf("Using device %d.\n", selectedDevice);

    return true;
}

__global__ void copyStringKernel(const char* src, char* dst) {
    int i = 0;

    while (src[i] != '\0') {
        dst[i] = src[i];
        ++i;
    }

    dst[i] = '\0';
}

int main() {
    if (!initCUDA()) {
        return EXIT_FAILURE;
    }

    const char* h_src = "HelloWorld from CUDA";
    const int size = static_cast<int>(strlen(h_src)) + 1;

    char* h_dst = static_cast<char*>(malloc(size));

    if (h_dst == nullptr) {
        fprintf(stderr, "Host memory allocation failed.\n");
        return EXIT_FAILURE;
    }

    char* d_src = nullptr;
    char* d_dst = nullptr;

    CUDA_CHECK(cudaMalloc((void**)&d_src, size));
    CUDA_CHECK(cudaMalloc((void**)&d_dst, size));

    CUDA_CHECK(cudaMemcpy(d_src, h_src, size, cudaMemcpyHostToDevice));

    copyStringKernel<<<1, 1>>>(d_src, d_dst);

    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize());

    CUDA_CHECK(cudaMemcpy(h_dst, d_dst, size, cudaMemcpyDeviceToHost));

    printf("Result: %s\n", h_dst);

    CUDA_CHECK(cudaFree(d_src));
    CUDA_CHECK(cudaFree(d_dst));

    free(h_dst);

    return EXIT_SUCCESS;
}
```

编译命令：

```bash
nvcc -O2 hello_world_full.cu -o hello_world_full
```

运行命令：

```bash
./hello_world_full
```

可能输出：

```text
Device 0: NVIDIA ...
Compute capability: ...
Using device 0.
Result: HelloWorld from CUDA
```

## 十、HelloWorld 示例中的 CUDA 执行流程

以上示例可以抽象为以下流程：

```text
1. CPU 执行 main 函数
2. CPU 查询 CUDA 设备
3. CPU 在 GPU 上分配显存
4. CPU 将输入数据复制到 GPU
5. CPU 启动 GPU kernel
6. GPU 执行 kernel
7. CPU 等待 GPU 执行完成
8. CPU 将 GPU 结果复制回主存
9. CPU 输出结果
10. CPU 释放 GPU 显存
```

对应的 CUDA API 如下：

| 步骤 | CUDA API 或语法 | 作用 |
|---|---|---|
| 查询设备数量 | `cudaGetDeviceCount` | 获取 CUDA 设备数量 |
| 查询设备属性 | `cudaGetDeviceProperties` | 获取设备名称和计算能力 |
| 选择设备 | `cudaSetDevice` | 指定当前使用的 GPU |
| 分配显存 | `cudaMalloc` | 在 device 端分配内存 |
| 数据传输 | `cudaMemcpy` | 在 host 和 device 之间复制数据 |
| 启动 kernel | `kernel<<<grid, block>>>` | 在 GPU 上执行 kernel |
| 检查 kernel 启动 | `cudaGetLastError` | 获取 kernel 启动错误 |
| 同步等待 | `cudaDeviceSynchronize` | 等待 GPU 执行完成 |
| 释放显存 | `cudaFree` | 释放 device memory |

## 十一、为什么 kernel 中的 `printf` 需要同步

GPU kernel 的启动通常是异步的。也就是说，CPU 启动 kernel 后，可能不会等待 GPU 立即执行完，而是继续执行后面的 host 代码。

如果程序写成：

```cpp
helloKernel<<<1, 1>>>();
return 0;
```

可能出现 GPU 端输出没有显示的情况。

因此应写成：

```cpp
helloKernel<<<1, 1>>>();
cudaDeviceSynchronize();
```

`cudaDeviceSynchronize()` 的作用是等待前面启动的 GPU 任务完成。对于使用 GPU `printf` 的 HelloWorld 程序，它还能确保 GPU 端输出缓冲被刷新。

## 十四、常见错误

### 1. 文件扩展名不是 `.cu`

错误情况：

```text
hello_world.cpp
```

如果文件中包含 kernel 启动语法：

```cpp
kernel<<<1, 1>>>();
```

普通 C++ 编译器无法识别。

正确做法：

```text
hello_world.cu
```

并使用：

```bash
nvcc hello_world.cu -o hello_world
```

---

### 2. 使用 `gcc` 或 `g++` 编译 CUDA 文件

错误命令：

```bash
g++ hello_world.cu -o hello_world
```

正确命令：

```bash
nvcc hello_world.cu -o hello_world
```

`nvcc` 能识别 CUDA 扩展语法，并负责把 host code 和 device code 分别交给合适的编译流程处理。

---

### 3. 忘记调用 `cudaDeviceSynchronize`

错误写法：

```cpp
helloKernel<<<1, 1>>>();
return 0;
```

可能导致 GPU 端 `printf` 不输出。

正确写法：

```cpp
helloKernel<<<1, 1>>>();
cudaDeviceSynchronize();
return 0;
```

---

### 4. kernel 启动参数写错

错误写法：

```cpp
helloKernel(1, 1);
```

正确写法：

```cpp
helloKernel<<<1, 1>>>();
```

CUDA kernel 启动必须使用三尖括号语法：

```cpp
<<<grid, block>>>
```

---

### 5. `__global__` 写错或遗漏

错误写法：

```cpp
void helloKernel() {
    printf("Hello from GPU!\n");
}
```

然后尝试：

```cpp
helloKernel<<<1, 1>>>();
```

这是错误的，因为普通函数不能用 kernel 启动语法调用。

正确写法：

```cpp
__global__ void helloKernel() {
    printf("Hello from GPU!\n");
}
```

---

### 6. 在不支持 CUDA 的机器上运行

如果没有 NVIDIA GPU、驱动未安装或 CUDA 环境配置不正确，程序可能报错。

可以先运行：

```bash
nvidia-smi
```

或：

```bash
nvcc --version
```

检查环境。

---

### 7. `cudaMemcpy` 方向写反

host 到 device：

```cpp
cudaMemcpy(d_src, h_src, size, cudaMemcpyHostToDevice);
```

device 到 host：

```cpp
cudaMemcpy(h_dst, d_dst, size, cudaMemcpyDeviceToHost);
```

方向写反会导致数据错误。

---

### 8. `cudaMalloc` 分配大小错误

错误写法：

```cpp
cudaMalloc((void**)&d_src, strlen(h_src));
```

字符串需要包含结尾的 `'\0'`。

正确写法：

```cpp
int size = strlen(h_src) + 1;
cudaMalloc((void**)&d_src, size);
```

---

### 9. host pointer 与 device pointer 混用

错误示例：

```cpp
copyStringKernel<<<1, 1>>>(h_src, h_dst);
```

kernel 在 GPU 上执行，不能直接使用普通 host 指针。

正确写法：

```cpp
copyStringKernel<<<1, 1>>>(d_src, d_dst);
```

---

### 10. 误以为 GPU 线程输出顺序固定

多个 GPU 线程同时执行时，`printf` 输出顺序不一定固定。即使线程编号是连续的，输出也可能交错。

因此不应根据 `printf` 的输出顺序判断线程执行顺序。

## 十五、回答模板

当用户询问 CUDA HelloWorld 程序时，可以按照以下方式回答：

```text
CUDA HelloWorld 的目的不是单纯输出字符串，而是展示 host 端如何启动 device 端 kernel。最小程序通常包含一个使用 __global__ 修饰的 kernel 函数，在 main 函数中通过 <<<grid, block>>> 启动该 kernel，并使用 cudaDeviceSynchronize() 等待 GPU 执行完成。这样可以观察 CPU 端代码和 GPU 端代码的执行关系。
```

当用户询问为什么要使用 `cudaDeviceSynchronize()` 时，可以回答：

```text
kernel 启动通常是异步的，host 端启动 kernel 后可能继续向下执行。如果程序很快结束，GPU 端的 printf 输出可能还没有完成。因此在 HelloWorld 示例中需要调用 cudaDeviceSynchronize()，等待 GPU 端 kernel 执行完成后再退出程序。
```

当用户询问 `<<<1, 1>>>` 的含义时，可以回答：

```text
<<<1, 1>>> 表示启动一个 grid，其中包含 1 个 block，每个 block 中包含 1 个 thread。因此这个 kernel 总共只启动 1 个 GPU 线程。它适合用于最小 HelloWorld 示例，但不能体现大规模并行能力。
```

当用户询问如何输出线程编号时，可以回答：

```text
可以在 kernel 中使用 blockIdx.x、blockDim.x 和 threadIdx.x 计算当前线程的全局编号。常见写法是 int global_id = blockIdx.x * blockDim.x + threadIdx.x。每个线程执行同一份 kernel 代码，但由于 global_id 不同，可以区分自己负责的数据位置。
```

## 十七、小结

CUDA HelloWorld 是学习 CUDA 编程的第一个实践程序。它通过最小 kernel 启动展示了 host 端与 device 端之间的调用关系，也让学习者第一次接触 `.cu` 文件、`__global__` 函数、`<<<grid, block>>>` 启动语法和 `cudaDeviceSynchronize()` 同步操作。

在掌握最小 HelloWorld 之后，可以进一步观察多个线程的编号输出，理解 block、thread 和全局线程编号的关系。随后可以加入 `cudaMalloc`、`cudaMemcpy` 和 `cudaFree`，完成更接近真实 CUDA 程序的 host-device 数据传输流程。

本节的核心结论是：CUDA HelloWorld 的重点不是输出字符串，而是理解 CPU 如何启动 GPU kernel，以及 GPU 中多个线程如何并行执行同一份 kernel 代码。
