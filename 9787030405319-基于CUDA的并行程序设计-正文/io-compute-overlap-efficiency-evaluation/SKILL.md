# I/O与计算时间重叠有效性评估

## 适用场景
- 系统支持异步I/O与计算重叠（如使用CUDA流、POSIX AIO、或MPI+I/O并行）
- 可独立测量I/O时间、计算时间和总运行时间
- 目标平台为核外计算系统、SSD存储平台或数据密集型科学计算环境

## 执行步骤
1. **获取关键时间指标**：
   - `io_time`：从外存读取/写入数据所耗时间（秒）
   - `compute_time`：CPU/GPU执行核心计算所耗时间（秒）
   - `total_time`：整个任务端到端运行时间（秒）

2. **比较I/O与计算时间接近程度**：
   - 若 `|io_time - compute_time|` 小于预设阈值（通常为两者较小值的10%~15%），则视为时间相近。

3. **验证总时间是否接近两者最大值**：
   - 检查 `total_time ≈ max(io_time, compute_time)`，允许少量调度或同步开销（通常不超过10%）。

4. **判定重叠有效性**：
   - 若同时满足上述两个条件，则I/O被有效掩盖，系统实现了高并行效率。
   - 否则，若 `total_time ≈ io_time` 且 `io_time >> compute_time`，则I/O成为瓶颈，未被有效覆盖。

5. **输出效率指标**（可选）：
   - 计算重叠效率：`efficiency = max(io_time, compute_time) / total_time`
   - 值越接近1，表示重叠效果越好。

> **注意**：本技能不适用于纯同步I/O模型或无流水线机制的系统。
