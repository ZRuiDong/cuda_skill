# 并行算法时间复杂度与耗费详细公式

## 核心定义

- **时间复杂度**：在PRAM（Parallel Random Access Machine）模型中，并行算法的时间复杂度定义为从第一个处理器开始执行到最后一个处理器完成所经历的最大并行计算步数（steps）。

- **并行耗费（Work）**：总计算量，等于时间复杂度乘以处理器数量 P。
  ```
  Work = Time_Complexity × P
  ```

## 经典案例：n个数求和

### 串行实现
- 执行步骤：n−1 次加法
- 时间复杂度：O(n)

### 并行实现（树形归约）
- 处理器数量：P = n/2（假设n为2的幂）
- 执行步骤：⌈log₂n⌉
- 时间复杂度：O(log₂n)
- 并行耗费：O(log₂n) × (n/2) = O(n log n)

### 性能对比
- 串行耗费：O(n)
- 并行耗费：O(n log n) > O(n) → 此并行方案在总计算量上不如串行高效
- 但若目标是降低运行时间（而非总计算量），则仍具价值

## 变量说明
| 变量 | 类型 | 描述 |
|------|------|------|
| n | integer | 输入问题规模（如数据元素数量） |
| P | integer | 并行处理器数量 |
| time_complexity | string | 时间复杂度表达式（如 O(log n)） |
| work | string | 并行耗费 = time_complexity × P |

## 注意事项
- PRAM模型假设所有处理器访问共享内存无延迟、无冲突
- 实际系统中需考虑：内存带宽限制、线程发散、通信延迟、负载不均衡等
- 本分析仅适用于理论评估，不可直接用于生产性能预测
