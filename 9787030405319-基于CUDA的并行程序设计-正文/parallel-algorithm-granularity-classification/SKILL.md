# 并行算法粒度分类

## 何时使用
- 正在设计并行算法并需要划分任务
- 优化CUDA核函数或GPU程序性能
- 需要在负载均衡与通信开销之间取得平衡

## 执行步骤
1. **评估子任务计算量**：估算每个子任务的浮点运算次数（computation_per_task）。
2. **评估通信开销**：估算任务间同步或数据传输所需时间（communication_overhead）。
3. **应用分类标准**：
   - **粗粒度**：计算量远大于通信开销；适用于大型循环或整体数据处理（如整幅图像滤波）。
   - **中粒度**：计算与通信相对平衡；适用于分块处理（如图像分块卷积）。
   - **细粒度**：计算量小，通信占比高；适用于像素级或语句级并行（如区域分解）。
4. **结合硬件调整**：
   - 在GPU上优先选择**细至中等粒度**以充分利用数千核心。
   - 确保每个线程有足够工作量以掩盖内存延迟，同时避免过粗导致流多处理器（SM）利用率低。
5. **输出推荐**：返回粒度级别（coarse / medium / fine）及对应的任务划分策略。

> 注意：粒度具有相对性，同一任务在不同硬件上可能归属不同级别。仅适用于可并行化问题。
