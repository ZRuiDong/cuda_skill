# 核外计算I/O瓶颈判定

## 何时使用
- 使用D0oC中间件执行核外计算
- 矩阵总大小 > 集群总可用内存（N × 单节点可用内存）
- 任务为I/O密集型（如稀疏矩阵向量乘SpMV）

## 如何执行
1. **确认核外模式**：若矩阵大小 > N × M（M为单节点实际可用内存），则必须使用外存，进入核外计算。
2. **监控性能表现**：
   - 核内计算性能应接近OpenMP基准（>90%）
   - 核外计算性能将趋近中间件外存访问带宽（如81–129 MB/s）
3. **验证I/O瓶颈**：
   - 测量实际I/O带宽
   - 若增加节点数后带宽不再提升（如趋近数据节点理论上限），表明I/O成为瓶颈
4. **实施优化**：
   - 减少节点数以降低I/O竞争
   - 确保每数据节点SSD带宽（如2 GB/s）被充分利用

> 注意：本技能不适用于计算密集型且数据可全缓存的场景；I/O瓶颈会导致总耗时t_total显著增加，进而推高CPU小时成本。
